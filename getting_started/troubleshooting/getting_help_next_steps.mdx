# Where to Get Help & Next Steps

This page directs you to a variety of resources that will help you deepen your understanding of GoogleTest and GoogleMock, connect with the community, and tackle advanced testing scenarios. Whether you are troubleshooting an issue or looking to expand your testing capabilities, these resources ensure you are supported throughout your journey.

## Official Documentation

- **Full Documentation Site:** Explore the complete GoogleTest and GoogleMock [documentation](https://google.github.io/googletest/) which contains tutorials, API references, user guides, and conceptual overviews.
- **API Reference:** Detailed descriptions of GoogleMock API elements, including [mock methods and classes](../api-reference/gmock-mocking-framework/mock-methods-and-classes), [expectations and actions](../api-reference/gmock-mocking-framework/expectations-actions-and-sequences), [matchers and cardinalities](../api-reference/gmock-mocking-framework/matchers-and-cardinalities), and [strictness modes and behaviors](../api-reference/gmock-mocking-framework/strictness-modes-and-behaviors).
- **Guides & Cookbooks:** Practical usage guides such as the [gMock Cookbook](../docs/gmock_cook_book.md), [Getting Started with GoogleMock](../getting_started/first_test_and_validation/using_googlemock_basics.mdx), and best practice tutorials.

## Community and Support

- **GoogleTest GitHub Repository:** Engage with the source code, report issues, and contribute [here](https://github.com/google/googletest).
- **Issue Tracker:** File detailed bug reports or feature requests via the GitHub issues page.
- **Discussion Forums:** Participate in discussions, ask questions, and share knowledge on platforms like Stack Overflow using the tag `#googletest`.
- **Mailing Lists & Groups:** Subscribe to mailing lists or groups dedicated to GoogleTest and related testing tools for announcements and community advice.

## Advanced Learning and Tools

- **Advanced Guides:** Learn about [mocking patterns](../guides/core-workflows/mocking-patterns.mdx), [parameterized tests](../guides/core-workflows/parameterized-tests.mdx), and [continuous integration strategies](../guides/real-world-scenarios/ci-integration.mdx).
- **Troubleshooting:** Find solutions to common installation and runtime issues in the [Common Installation Issues](../getting_started/troubleshooting/common_install_issues.mdx) and [Unexpected Test Behavior & Debugging](../faq/troubleshooting-optimization/unexpected-test-behavior.mdx) pages.
- **Integration Help:** Guidance for integrating with build systems like Bazel and CMake, and IDEs such as Visual Studio and VS Code.
