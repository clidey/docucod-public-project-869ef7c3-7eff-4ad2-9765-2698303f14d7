# Related Projects and Extensions

Explore noteworthy community-driven and Google-sponsored projects that extend the functionality and usability of GoogleTest beyond its core capabilities. These tools enhance test automation, visualization, integration, and reporting, helping you adapt GoogleTest to diverse environments and workflows.

---

## Overview

GoogleTest's modular design and widespread adoption have fostered a vibrant ecosystem of complementary projects. Whether you need a graphical test runner, integration with Continuous Integration (CI) environments, or advanced test result listeners, this collection will provide solutions to enrich your testing framework.

This documentation highlights popular projects, outlines their primary benefits, and gives guidance on extending or customizing GoogleTest to fit your specific development needs.

---

## Popular External Projects

### GTest Runner

A Qt5-based automated test runner with a graphical user interface that works on Windows and Linux. GTest Runner offers:

- Test progress visualization through progress bars
- Detailed reporting of test failures
- User-friendly browsing of tests and outcomes

Visit the project repository for download and usage instructions: [GTest Runner](https://github.com/nholthaus/gtest-runner)

### GoogleTest UI

A lightweight test runner built in C# providing a rich UI for Windows users. Key features:

- Runs your compiled test binary and monitors its execution
- Displays test progress and failure details
- Allows direct interaction with failed tests to review logs

Explore the project here: [GoogleTest UI](https://github.com/ospector/gtest-gbar)

### GTest TAP Listener

An event listener compatible with the Test Anything Protocol (TAP), enabling GoogleTest to output test results in TAP format. This is particularly useful for integrating into test runners or reporting tools that consume TAP.

Learn more at: [GTest TAP Listener](https://github.com/kinow/gtest-tap-listener)

### gtest-parallel

Runs GoogleTest tests in parallel from your existing test binary, greatly speeding up test suite execution, especially beneficial for large test collections.

Details and setup are available at: [gtest-parallel](https://github.com/google/gtest-parallel)

### Visual Studio & VS Code Extensions

* **GoogleTest Adapter**: Visual Studio Code extension allowing you to view and run GoogleTest tests in a tree structure, set breakpoints, and debug.
  - [GoogleTest Adapter](https://marketplace.visualstudio.com/items?itemName=DavidSchuldenfrei.gtest-adapter)

* **C++ TestMate**: Another VS Code extension supporting GoogleTest and other C++ testing frameworks for graphical test discovery and debugging.
  - [C++ TestMate](https://github.com/matepek/vscode-catch2-test-adapter)

### Cornichon

A small DSL parser that helps generate boilerplate test code for GoogleTest using Gherkin syntax.

Find the project at: [Cornichon](https://pypi.org/project/cornichon/)

## Extending and Adapting GoogleTest

GoogleTest is designed to be adaptable to diverse development environments. To incorporate or extend it effectively:

- **Embed GoogleTest in your build system:** Use CMake's `add_subdirectory()` or add as a git submodule for tight integration and consistent compiler/linker settings.
- **Customize test runners:** Projects like GTest Runner or GoogleTest UI can serve as bases or inspiration to tailor test execution interfaces.
- **Leverage TAP listeners:** For environments relying on TAP, the `gtest-tap-listener` project ensures interoperability.
- **Speed up with parallelism:** Adopt `gtest-parallel` to reduce feedback cycles by running tests simultaneously.
- **Enhance IDE support:** Utilize or contribute to VS and VS Code extensions for better developer productivity.

When tailoring GoogleTest to your workflow, be mindful of:

- Compatibility with your build and CI systems
- Managing external dependencies gracefully
- Keeping test discovery and reporting seamless

## Real-World Scenarios

- **CI Integration:** Use `gtest-parallel` together with TAP listeners to accelerate test runs and produce standardized output for Jenkins, Travis CI, or GitHub Actions.
- **Debugging Failures:** Employ GoogleTest UI or VS Code extensions to quickly navigate logs and test failures without leaving the IDE.
- **Cross-Platform Testing:** Use Qt-based runners like GTest Runner to provide consistent test visualization on Windows and Linux.
