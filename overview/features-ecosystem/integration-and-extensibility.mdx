# Integration & Extensibility

Explore how GoogleTest seamlessly fits into your existing development environment and testing workflows. This guide highlights compatibility with popular CI platforms, build systems like CMake and Bazel, as well as extension points that enable you to tailor and expand the framework. Discover upstream and downstream integrations with tools such as Abseil, TAP listeners, and VS Code test adapters to build a cohesive test ecosystem.

---

## Continuous Integration Compatibility

GoogleTest is designed to integrate smoothly with a wide range of Continuous Integration (CI) platforms, ensuring automated, reliable test execution in your build pipelines.

- **CI Platforms Supported:** Jenkins, GitHub Actions, GitLab CI, CircleCI, Travis CI, and more.
- **Result Reporting:** Supports common test result formats (JUnit XML, JSON) enabling easy consumption by CI systems.
- **Built-In Flags:** Customize test execution through command-line flags optimized for CI usage.

**Tip:** Use GoogleTest's XML output (`--gtest_output=xml:<file>`) in your CI to generate detailed, structured reports compatible with your CI dashboards.

## Build System Integration

GoogleTest supports popular C++ build systems, providing ready-to-use integration patterns.

### CMake

- Official `CMakeLists.txt` files allow you to include GoogleTest and GoogleMock as submodules or via package managers.
- Targets such as `gtest` and `gmock` can be linked easily, simplifying your project setup.

Example snippet to add GoogleTest as a subdirectory:

```cmake
# Add GoogleTest
add_subdirectory(path/to/googletest)

# Link your test executable with GoogleTest libraries
add_executable(my_tests test_main.cpp ...)
target_link_libraries(my_tests gtest_main gmock)

# Enable testing
enable_testing()
add_test(NAME my_tests COMMAND my_tests)
```

### Bazel

- GoogleTest is fully supported in Bazel builds.
- It integrates via `cc_test` and supports `googletest` rules.

Example usage:

```python
cc_test(
    name = "my_tests",
    srcs = ["my_tests.cc"],
    deps = ["@com_google_googletest//:gtest_main"],
)
```

**Recommendation:** Keep GoogleTest as an external dependency managed by workspace rules for consistent versioning.

## Extension Points

GoogleTest is extensible, allowing you to customize and extend its capabilities:

- **Event Listeners:** Capture test events and extend reporting with custom output formats or integrations.
- **Custom Assertions:** Implement custom assertion macros to handle domain-specific checks.
- **Environment Setup:** Override global setup and teardown through `::testing::Environment`.
- **Matchers & Actions:** Extend GoogleMock with your own matchers and actions to write more expressive mocks.

Example: Adding a simple event listener to log test start and end:

```cpp
class SimpleListener : public ::testing::EmptyTestEventListener {
 public:
  void OnTestStart(const ::testing::TestInfo& test_info) override {
    std::cout << "Starting: " << test_info.test_case_name() << "." 
              << test_info.name() << std::endl;
  }
  void OnTestEnd(const ::testing::TestInfo& test_info) override {
    std::cout << "Finished: " << test_info.test_case_name() << "." 
              << test_info.name() << std::endl;
  }
};

int main(int argc, char** argv) {
  ::testing::InitGoogleTest(&argc, argv);

  ::testing::TestEventListeners& listeners =
      ::testing::UnitTest::GetInstance()->listeners();

  listeners.Append(new SimpleListener());

  return RUN_ALL_TESTS();
}
```

## Upstream and Downstream Integrations

GoogleTest and GoogleMock integrate smoothly with complementary tooling ensuring extensible test ecosystems.

- **Abseil:** A foundational library from Google that works well with GoogleTest.
- **Test Anything Protocol (TAP):** GoogleTest can feed results to TAP listeners for compatibility with TAP-based infrastructures.
- **VS Code Test Adapters:** Enable seamless test discovery, execution, and reporting within the editor.

These integrations provide flexibility in how you consume and manage test results across environments.

GoogleTest runs on a wide range of platforms and compilers:

- **Operating Systems:** Linux, macOS, Windows, FreeBSD, and embedded platforms.

## Troubleshooting and Best Practices

- When integrating into CI, verify XML or JSON output correctness.
- Take advantage of GoogleTest's command-line options to filter and run tests selectively in different environments.
- Prefer using GoogleMock's extensibility features for mocking complex dependencies.
