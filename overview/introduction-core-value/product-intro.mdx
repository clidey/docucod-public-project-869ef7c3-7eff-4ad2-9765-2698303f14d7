# What is GoogleTest?

## Uniting C++ Testing and Mocking Under One Trusted Framework

GoogleTest is Google's premier C++ test framework that combines the powerful capabilities of the original GoogleTest for unit testing and GoogleMock for mocking into a unified testing toolkit. Trusted by industry-leading projects such as Chrome, LLVM, Protocol Buffers, and OpenCV, GoogleTest enables developers to write reliable, maintainable, and automated tests that accelerate software quality and developer confidence.

### Why GoogleTest Matters to You

- **Reliable Tests with Rich Expression:** GoogleTest helps you express test assertions clearly and concisely, catching bugs before they reach production.
- **Integrated Mocking with GoogleMock:** Seamlessly define mock classes for interfaces and control their behavior for interaction-based testing.
- **Wide Industry Adoption:** Be part of a trusted ecosystem where critical software depends on its dependability.
- **Streamlined Testing Workflow:** Efficiently write, run, and maintain tests with meaningful diagnostics and clear failure contexts.

### Who Should Use GoogleTest?
GoogleTest is designed for C++ developers, software engineers, and QA professionals aiming to embed automated testing and mocking directly into their development process to ensure robust, high-quality software.

## What is GoogleTest?

GoogleTest is a comprehensive C++ testing and mocking framework developed and maintained by Google. It fuses the strengths of two formerly separate tools — GoogleTest, focused on unit testing with powerful assertion capabilities, and GoogleMock, a robust mocking framework — into a single, cohesive product.

### Core Purpose

This toolkit exists to empower developers to write tests that are both expressive and precise, enabling:

- Verification of the correctness of code logic through structured tests.
- Creation of mock objects that simulate and verify interactions with dependencies.
- Ordering, sequencing, and cardinality control over expected method calls.
- Automated verification of expectations without manual record-and-replay hassles.

### What Sets GoogleTest Apart

- **Unified Framework:** Both testing and mocking are accessible through one consistent API and tooling experience.
- **Industry-Proven:** Used extensively in Google's flagship products, proving its robustness and scalability.
- **Intuitive Expectations:** A domain-specific language for setting up mock expectations that reads almost like English.
- **Automatic Verification:** Immediate failure reporting, facilitating quick identification of incorrect behaviors.

## Key Features & Capabilities of GoogleTest

GoogleTest provides an array of tools to achieve comprehensive testing and mocking coverage for your C++ projects.

### Essential Testing Features

- **Rich Assertions:** A vast set of assertions ranging from simple equality checks to complex container and predicate-based validations.
- **Test Fixtures:** Organize common setup and teardown code for test groups.
- **Parameterized Tests:** Write tests that run over a range of inputs effortlessly.

### Integrated Mocking via GoogleMock

- **Declarative Mock Classes:** Use the `MOCK_METHOD` macro to mock virtual functions, even those with complex signatures and qualifiers.
- **Flexible Expectations:** Specify how many times a method should be called, with what arguments, in which order, and what it should return or do.
- **Matchers and Actions:** Use expressive matchers to validate argument properties and specify actions including returning values, invoking callbacks, or modifying arguments.
- **Sequencing and Ordering:** Control call order with `InSequence`, `After` clauses, and sequences for partial or total order verification.
- **Strict, Nice, and Naggy Mocks:** Customize mock behavior regarding uninteresting calls with modes that suppress warnings, report errors, or warn harmlessly.

#### Concrete Example: Defining a Mock Class

```cpp
#include <gmock/gmock.h>

class Foo {
 public:
  virtual ~Foo() {}
  virtual int GetSize() const = 0;
  virtual std::string Describe(const char* name) = 0;
  virtual bool Process(int elem, int count) = 0;
};

class MockFoo : public Foo {
 public:
  MOCK_METHOD(int, GetSize, (), (const, override));
  MOCK_METHOD(std::string, Describe, (const char* name), (override));
  MOCK_METHOD(bool, Process, (int elem, int count), (override));
};
```

### Specifying Expectations and Behaviors

```cpp
using ::testing::Return;
using ::testing::_;

TEST(FooTest, ProcessBehavior) {
  MockFoo mock_foo;

  // Set default action
  ON_CALL(mock_foo, GetSize()).WillByDefault(Return(5));

  // Expect Describe to be called exactly once and return "Test"
  EXPECT_CALL(mock_foo, Describe("example"))
      .Times(1)
      .WillOnce(Return("Test"));

  // Exercise mock
  EXPECT_EQ(5, mock_foo.GetSize());
  EXPECT_EQ("Test", mock_foo.Describe("example"));
}
```

### Testing Flow and Verification

- When a mock object goes out of scope, GoogleMock automatically verifies that all expectations were satisfied.
- Unexpected calls or unfulfilled expectations will result in informative errors pointing precisely to the cause.
- Default actions ensure smooth handling of calls that are not explicitly expected.

## Why Should You Care?

Leveraging GoogleTest will transform your software testing experience by enabling:

- **Faster Bug Detection:** Catch defects early with precise, clear expectations.
- **Better Code Design:** Mocks force clearer separation of concerns by encouraging interface-based design.
- **Robust and Maintainable Tests:** Handle complex interactions gracefully without fragile test code.
- **Productivity:** Cut testing effort by automating verification and using rich DSLs for mocking.

Before GoogleTest, writing mocks or complex tests in C++ was error-prone and tedious. Now you have a mature, battle-tested framework that scales with your project's complexity.

## Getting Started Preview

Starting with GoogleTest is straightforward:

1. Include the library bundled with the [googletest project](https://github.com/google/googletest).
2. Define your mock classes using the `MOCK_METHOD` macro.
3. Use `EXPECT_CALL` and `ON_CALL` to specify behaviors and expectations.
4. Write your test logic calling into mocked objects.
5. Run tests regularly and leverage GoogleTest's rich output for diagnosis.

**Prerequisites:** Familiarity with C++ virtual methods, basic unit testing concepts.

For a deeper dive into mocking syntax, explore the [Mocking Reference](docs/reference/mocking.md) and [gMock Cheat Sheet](docs/gmock_cheat_sheet.md).
