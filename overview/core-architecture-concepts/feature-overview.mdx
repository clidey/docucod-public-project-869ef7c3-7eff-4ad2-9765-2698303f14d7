# Feature Highlights: What You Can Do

Discover the core features and capabilities of GoogleTest and GoogleMock — engineered to empower your C++ testing workflows with flexibility, precision, and robustness. This page walks you through the framework’s most important functionalities, from dynamic test discovery to advanced mocking techniques, helping you understand exactly how GoogleTest and GoogleMock enable efficient, scalable, and maintainable unit and integration testing.

---

## A Powerful, Flexible Testing Framework Tailored for C++

GoogleTest and GoogleMock together form a comprehensive testing framework designed for C++, adhering to and extending the well-known xUnit architecture. With a focus on automated test discovery, expressive assertions, and rich mocking support, this framework enables you to write tests that are both easy to maintain and powerful enough to capture complex behaviors.

> Whether you’re validating a small module or stitching together an intricate system of interacting components, these features elevate your testing strategy.

### Key Features at a Glance

- **xUnit Compatibility:** Seamlessly fits into common testing workflows familiar to developers from other languages and frameworks.
- **Dynamic Test Discovery:** Automatically detects test cases and test suites at runtime, sparing you from manual registration and enabling rapid test iteration.
- **Flexible Assertions:** Leverage a variety of assertion types, including fatal and non-fatal assertions, to clearly specify expected outcomes and handle failures gracefully.
- **Advanced Mocking Capabilities:** High-fidelity mocking with GoogleMock components like `EXPECT_CALL`, `ON_CALL`, custom matchers, and actions.
- **Parameterized and Typed Tests:** Run the same test logic over multiple input data sets or types, dramatically increasing test coverage with minimal code duplication.
- **Sophisticated Runner Options:** Multiple ways to run and filter tests, enabling precise control over your testing process.

This suite is designed for C++ developers, testers, and teams aiming to boost code quality and reliability through rigorous automated testing.

## What Can You Do with GoogleTest and GoogleMock?

### 1. Write Structured Tests That Run Automatically

Define individual tests organized into test cases and suites, and let GoogleTest dynamically find and run them without extra boilerplate. This simplifies the testing lifecycle and accelerates development feedback.

### 2. Use Expressive Assertions to Define Precise Conditions

From simple truth checks to detailed value comparisons, GoogleTest offers assertions that clearly express your expectations. You can choose between fatal assertions (which abort the current test on failure) and non-fatal assertions (which allow the test to continue), tailoring your validation strategy.

### 3. Create Mock Objects With Full Control Over Interactions

GoogleMock lets you define mock classes declaratively using `MOCK_METHOD` macros and then specify exactly how those mocks should behave and be called using an intuitive syntax:

- Set **expectations** on calls with `EXPECT_CALL`, constraining the number of calls, argument values, order, and sequences.
- Define **default behaviors** using `ON_CALL`, without asserting the call must happen.
- Leverage **custom matchers** and **complex predicates** to finely tune argument matching.
- Chain multiple **actions** to simulate side-effects, return values, or delegate calls.

You can precisely assert and control how your components interact by setting cardinalities (`Times`), sequences (`InSequence`, `After`), and call ordering.

### 4. Run Parameterized and Typed Tests to Maximize Coverage

Instead of duplicating similar tests for different inputs or types, parameterized tests allow you to define a single test template and run it across various data sets or types effortlessly, enabling comprehensive validation with less code.

### 5. Handle Special Cases Gracefully

You can treat unexpected calls, uninteresting calls, and excessive calls separately, deciding whether to ignore, warn, or fail. With `NiceMock`, `NaggyMock`, and `StrictMock` wrappers, you control the strictness of your mocks’ behaviors, improving test clarity and stability.

## Feature Details and User Scenarios

### Dynamic Test Discovery and xUnit Compliance

GoogleTest scans your test code and automatically identifies all tests without you manually registering each one. This lets you focus on writing your test logic rather than test infrastructure.

A simple test example:

```cpp
TEST(FooTest, HandlesTrueInput) {
  EXPECT_TRUE(MyFunction(true));
}
```

Once compiled, your test binary knows about this and other tests and runs them via the command line or IDE, supporting filters and fine-grained control.

### Flexible Assertions

GoogleTest supports multiple assertion macros, such as `EXPECT_EQ`, `ASSERT_TRUE`, `EXPECT_THAT` (which supports matchers) and distinguishes between fatal failures (stop test execution) and non-fatal failures (allow continuing tests). Use this to create clear, maintainable test conditions.

### Comprehensive Mocking Support

Mocking is integral for testing how components interact, especially when dealing with dependencies or side-effects.

**How to use:**

- Define a mock class using `MOCK_METHOD`:

```cpp
class MockFoo {
 public:
  MOCK_METHOD(bool, Bar, (int x, int y));
};
```

- Set expectations:

```cpp
MockFoo mock;
EXPECT_CALL(mock, Bar(5, _)).WillOnce(Return(true));
```

- When the mocked method is called during test execution, GoogleMock verifies the arguments, call order, and invocation counts according to your expectations.

**Advanced features include:**
- Matching multiple arguments together with `.With()`.
- Setting cardinalities with `.Times()`, like `Exactly(3)` or `AtLeast(1)`.
- Specifying call sequences using `.InSequence()` and `.After()`.
- Providing default actions with `ON_CALL`.
- Combining multiple actions in sequences of `.WillOnce()` and `.WillRepeatedly()`.

### Parameterized and Typed Tests

Run the same test body with different inputs:

```cpp
class MyTest : public ::testing::TestWithParam<int> {};

TEST_P(MyTest, HandlesInput) {
  int n = GetParam();
  EXPECT_TRUE(CheckValue(n));
}

INSTANTIATE_TEST_SUITE_P(
  MyInputs,
  MyTest,
  ::testing::Values(1, 2, 3, 5, 8));
```

Typed tests run a test template over multiple types.

### Controlling Mock Object Strictness

Decide how strict GoogleMock should be about unexpected or uninteresting calls:

- `NiceMock`: Suppresses warnings on uninteresting calls, useful for ignoring irrelevant behavior.
- `NaggyMock`: Prints warnings on uninteresting calls (default behavior).
- `StrictMock`: Treats uninteresting calls as errors, useful when you want tests to be very precise.

### Powerful Runner and Test Filtering

Run your tests selectively using runner options, filters, and more. This boosts testing efficiency during development iterations.

## Why This Matters to You

GoogleTest and GoogleMock’s features empower you to write tests that are:

- **Maintainable:** Structured, clear, and concise, making tests easy to read and update.
- **Reliable:** Verify not only outcomes but also interactions between components.
- **Efficient:** Automated discovery and rich assertion options accelerate feedback cycles.
- **Scalable:** Parameterized tests support broad coverage without code duplication.

Imagine transforming your test suite to immediately expose interaction defects, understand the exact failure points through expressive diagnostic messages, and run precise subsets of tests driven by your current focus—all while writing less code.
