# Parameterized and Typed Tests

GoogleTest supports advanced testing techniques using parameterized and typed tests. These methods enable efficient, data-driven testing by running the same logical test code across multiple input values or C++ types, thereby achieving broad coverage with minimal duplication.

---

## Workflow Overview

**Purpose:**
This guide introduces you to the use of value-parameterized tests and type-parameterized tests in GoogleTest. By following these techniques, you can write flexible test suites that systematically vary input values or type information to verify behavior comprehensively.

**Prerequisites:**
- Familiarity with writing basic GoogleTest tests (e.g., using `TEST` and `TEST_F`).
- GoogleTest installed and configured in your C++ project.
- Basic understanding of C++ templates and types.

**Expected Outcome:**
After completing this guide, you will be able to:
- Define value-parameterized tests that run the same test logic over diverse input values.
- Use type-parameterized tests to run tests across multiple C++ types.
- Organize and instantiate these tests effectively with GoogleTest macros.

**Time Estimate:** 15–30 minutes, depending on prior experience.

**Difficulty Level:** Intermediate.

## Value-Parameterized Tests: Testing Across Input Values

### What Are Value-Parameterized Tests?
Value-parameterized tests run the same test body repeatedly with different input values, enabling a single test suite to cover multiple scenarios.

### How to Define a Value-Parameterized Test Suite

1. **Create a Test Fixture Class that Inherits from `testing::TestWithParam<T>`**

```cpp
#include <gtest/gtest.h>

class MyValueTest : public testing::TestWithParam<int> {
  // Optional setup/teardown or helpers
};
```

Here, `int` is the type of the parameter.

2. **Define the Parameterized Test with `TEST_P` Macro**

```cpp
TEST_P(MyValueTest, IsEven) {
  int n = GetParam();  // Access the parameter value
  EXPECT_EQ(n % 2, 0);
}
```

3. **Instantiate the Test Suite with a Set of Values**

```cpp
INSTANTIATE_TEST_SUITE_P(
    EvenNumbers,         // Instance name
    MyValueTest,         // Test suite name
    testing::Values(2, 4, 6, 8)  // Parameter values
);
```

### Running
Each value in the `Values()` call will create a separate test instance, reported distinctly during test runs.

### Accessing Parameters in Tests
- Use `GetParam()` inside `TEST_P` to access the current input value.

### Parameter Generators
GoogleTest also offers generators to create input sets:
- `Values(v1, v2, ...)`
- `ValuesIn(container)`
- `Range(begin, end [, step])`
- `Bool()`
- `Combine(g1, g2, ..., gN)` (Cartesian product of multiple generators)

### Example: Testing a Function with Multiple Inputs

```cpp
TEST_P(MyValueTest, IsPositive) {
  int n = GetParam();
  EXPECT_GT(n, 0);
}

INSTANTIATE_TEST_SUITE_P(
    PositiveNumbers,
    MyValueTest,
    testing::Values(1, 2, 3, 5, 8, 13)
);
```

## Type-Parameterized Tests: Testing Across C++ Types

### What Are Type-Parameterized Tests?
Type-parameterized tests enable running the *same* test code with different C++ types—for example, testing a template class with different type arguments.

### Defining a Typed Test Suite

1. **Define a Test Fixture Class Template Parameterized by the Type**

```cpp
template <typename T>
class MyTypedTest : public testing::Test {
 public:
  T value;
};
```

2. **Declare the List of Types**

```cpp
using MyTypes = testing::Types<int, double, std::string>;
```

3. **Associate the Types with the Test Suite:**

```cpp
TYPED_TEST_SUITE(MyTypedTest, MyTypes);
```

4. **Define Tests with `TYPED_TEST`**

```cpp
TYPED_TEST(MyTypedTest, IsDefaultConstructible) {
  TypeParam val = TypeParam();
  (void)val;  // Use val to avoid unused warnings
  SUCCEED();
}
```

The macro uses `TypeParam` to represent the current type under test.

### Defining Type-Parameterized Tests without Knowing Types (Advanced)

Sometimes you want to define reusable test *patterns* without specifying types initially. To do this:

1. Use `TYPED_TEST_SUITE_P` to declare the parametric test suite.
2. Define tests with `TYPED_TEST_P`.
3. Register the tests with `REGISTER_TYPED_TEST_SUITE_P`.
4. Instantiate with actual types using `INSTANTIATE_TYPED_TEST_SUITE_P`.

Example:

```cpp
// Declare pattern
TYPED_TEST_SUITE_P(MyPattern);

TYPED_TEST_P(MyPattern, DoesSomething) {
  TypeParam val = TypeParam();
  SUCCEED();
}

// Register the test
REGISTER_TYPED_TEST_SUITE_P(MyPattern, DoesSomething);

// Instantiate with types
using SomeTypes = ::testing::Types<int, float>;
INSTANTIATE_TYPED_TEST_SUITE_P(MyInstance, MyPattern, SomeTypes);
```

## Practical Tips and Best Practices

- **Use Typed Tests for Template Classes or Functions** when you want to validate behavior across multiple types without duplicating code.

- **Use Value-Parameterized Tests** when verifying functions or methods against various inputs.

- Avoid overly specific parameter matching, as that can make tests brittle.

- Use `INSTANTIATE_TEST_SUITE_P` names clearly to distinguish different parameter or type combinations.

- For better test names in type-parameterized tests, provide a custom name generator class to `TYPED_TEST_SUITE` and `INSTANTIATE_TYPED_TEST_SUITE_P`.

- Combine parameter generators with `Combine()` for multidimensional parameter spaces.

- Remember that `GetParam()` is only available within `TEST_P` tests.

- Typed test fixture classes can have static members and helper functions that depend on the template parameter.

- To avoid instantiation errors, ensure your test code compiles for all types in the type list.

## Example Usage of Value-Parameterized Tests

```cpp
#include <gtest/gtest.h>

class IsEvenTest : public testing::TestWithParam<int> {
};

TEST_P(IsEvenTest, CheckIsEven) {
  int n = GetParam();
  EXPECT_EQ(n % 2, 0);
}

INSTANTIATE_TEST_SUITE_P(EvenNumbers,
                         IsEvenTest,
                         testing::Values(2, 4, 6, 8));
```

Results in:
- Test instances for input values 2, 4, 6, 8 running `CheckIsEven`.

## Example Usage of Typed Tests

```cpp
#include <gtest/gtest.h>

template <typename T>
class NumericLimitsTest : public testing::Test {
};

using NumericTypes = testing::Types<int, float, double>;
TYPED_TEST_SUITE(NumericLimitsTest, NumericTypes);

TYPED_TEST(NumericLimitsTest, IsNonNegative) {
  EXPECT_GE(std::numeric_limits<TypeParam>::min(), 0);
}
```

This runs `IsNonNegative` with `int`, `float`, and `double`.

## Troubleshooting

- **Missing `GetParam()` or Wrong Fixture Base Class:**
  - If `GetParam()` is not available, confirm your fixture class inherits from `testing::TestWithParam<T>`.

- **No Test Instantiations Run:**
  - Ensure you invoke `INSTANTIATE_TEST_SUITE_P` for your parameterized test suite.
  - Confirm parameters are valid and your test filter includes the test names.

- **Compilation Errors with Typed Tests:**
  - Verify that your test code compiles with *all* types in the `Types` list.
  - Consider providing explicit type conversion or simplifying the tests.

- **Test Names Not Distinguishable:**
  - Use custom test name generators for clear output.

- **Overly Strict Expectations in Parameterized Tests:**
  - Avoid specifying exact values unless necessary to keep tests maintainable.
