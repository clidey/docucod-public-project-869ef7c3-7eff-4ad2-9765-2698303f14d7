# Where to Learn More & Get Help

This page gathers essential resources to help you deepen your knowledge of GoogleTest and GoogleMock, find official documentation, engage with the community, and explore advanced guides and examples. Use this as your hub for expanding expertise beyond the basics and resolving complex questions.

---

## Official Documentation

### Core User Guide
- **GoogleTest User's Guide**: Explore the full [User's Guide](https://github.com/google/googletest/blob/main/docs/index.md) that covers writing tests, assertions, mock usage, and more.
- **GoogleMock for Dummies**: Start with the beginner-friendly [gMock for Dummies](https://github.com/google/googletest/blob/main/docs/gmock_for_dummies.md) to understand mocking basics and workflows.
- **Mocking Reference**: Get a detailed list of mock-related macros, classes, and usage patterns in the [Mocking Reference](https://github.com/google/googletest/blob/main/docs/reference/mocking.md).
- **gMock Cookbook**: Access practical recipes, design tips, and solutions for common mocking scenarios in the [gMock Cookbook](https://github.com/google/googletest/blob/main/docs/gmock_cook_book.md).
- **gMock FAQ**: Find answers to frequent questions and troubleshooting advice in the [Legacy gMock FAQ](https://github.com/google/googletest/blob/main/docs/gmock_faq.md).

### Reference Documentation
- **Matchers Reference**: Understand argument matchers and how to compose them in the [Matchers Reference](https://github.com/google/googletest/blob/main/docs/reference/matchers.md).
- **Actions Reference**: Learn about specifying mock method behaviors through the [Actions Reference](https://github.com/google/googletest/blob/main/docs/reference/actions.md).
- **Cardinalities and Call Expectations**: Details on call count expectations and timing control are in [Cardinalities Reference](https://github.com/google/googletest/blob/main/api-reference/mock-configuration-behavior/cardinalities-reference.md).

## Tips to Get the Most Help

- Always search existing documentation and issues before opening new questions.
- Use verbose flags like `--gmock_verbose=info` to gain insights when running tests.
- Structure your questions with code samples and error messages for community support.
- Consider using `NiceMock` when you are not interested in uninteresting call warnings, and `StrictMock` when you want to enforce strict call verifications.
