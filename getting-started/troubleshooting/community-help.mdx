# Where to Find Help

If you encounter challenges or have questions while using GoogleTest, this guide directs you to the best channels for community support, issue reporting, and extensive documentation. Knowing where to get help promptly empowers you to maintain smooth development and testing workflows.

---

## Community Support Channels

Engaging with the GoogleTest community often yields fast and practical advice. Here are the primary avenues where you can ask questions, discover examples, and connect with fellow users:

- **GitHub Discussions and Issues**
  - Report bugs, request features, or ask for help on the official [GoogleTest GitHub repository](https://github.com/google/googletest).
  - Before opening an issue, verify that it hasn't been reported already.

- **Stack Overflow**
  - Search or post questions tagged with [`googletest`](https://stackoverflow.com/questions/tagged/googletest).
  - Community members including GoogleTest contributors often provide detailed answers.

- **GoogleTest Community-Created Documentation**
  - Helpful user-generated guides like the [Googlemock Insights](https://github.com/ElectricRCAircraftGuy/eRCaGuy_dotfiles/blob/master/googletest/insights.md) provide additional perspectives and tips.
  - This resource is great for deep dives and practical usage insights from experienced developers.

- **Mailing Lists or Chat**
  - Although GoogleTest does not have an official mailing list, some open-source communities and projects embed GoogleTest discussions on platforms such as Google Groups or IRC-based channels. Check the repository README or community links for updates.

## Issue Reporting Best Practices

When reporting issues or bugs, making your report clear and reproducible maximizes the chance of a quick resolution:

1. **Describe the Problem Clearly:**
   - What is the expected behavior?
   - What is the actual behavior?

2. **Provide a Minimal Reproducible Example:**
   - Include a small code snippet that reliably reproduces the issue.

3. **Environment Details:**
   - Operating system and version.
   - Compiler type and version.
   - GoogleTest version.

4. **Steps to Reproduce:**
   - Detailed instructions to reproduce the problem.

5. **Error Messages or Logs:**
   - Any compiler errors, runtime failures, or test output.

Submit issues here: [https://github.com/google/googletest/issues](https://github.com/google/googletest/issues)

## Tips for Efficient Help-Seeking

- Search existing issues and questions before posting new ones.
- Provide detailed context to save time for both you and helpers.
- Include minimal but complete code snippets to illustrate your problem.
- Refer to the exact GoogleTest version you are using.
