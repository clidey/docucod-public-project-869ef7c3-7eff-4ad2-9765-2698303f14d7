# Getting Help & Community Resources

When you are working with GoogleTest and GoogleMock, getting timely help and connecting with the community is critical to your success. This page directs you to official resources, community forums, FAQs, and ways you can contribute questions or report issues — helping you keep moving forward confidently.

---

## Official Documentation and User Guides

GoogleTest maintains comprehensive documentation designed to support users at all skill levels. These resources include:

- **[GoogleTest User's Guide](https://google.github.io/googletest/)**: A well-structured entry point into GoogleTest’s features, concepts, and best practices.
- **[GoogleTest Primer](https://google.github.io/googletest/primer.html)**: A beginner-friendly guide to writing your first tests.
- **[Advanced Guides](https://google.github.io/googletest/advanced.html)**: For mastering complex assertions, fixtures, death tests, and mocks.
- **[API Reference](https://google.github.io/googletest/reference/assertions.html)**: Detailed references for all GoogleTest and GoogleMock APIs.

You’ll find these links at the top of the main documentation or in navigation menus across the site.

## Community Forums and Discussion

Engaging with the community is an excellent way to solve problems, learn from peers, and share your experiences.

- **GitHub Issues**: [https://github.com/google/googletest/issues](https://github.com/google/googletest/issues) — Report bugs or request features. Before posting, search for existing issues to avoid duplicates.

- **Stack Overflow**: Use the tag [`googletest`](https://stackoverflow.com/questions/tagged/googletest) to ask questions and explore solved problems.

- **Community-Created Documentation**: Users have contributed valuable resources such as [Googlemock Insights](https://github.com/ElectricRCAircraftGuy/eRCaGuy_dotfiles/blob/master/googletest/insights.md) which offer deeper practical understanding.

- **Related Open Source Projects**: Explore tools around GoogleTest like test runners and IDE extensions that the community builds:
  - [GTest Runner GUI](https://github.com/nholthaus/gtest-runner)
  - [GoogleTest Adapter for VS Code](https://marketplace.visualstudio.com/items?itemName=DavidSchuldenfrei.gtest-adapter)

## Reporting Issues and Contributing

GoogleTest welcomes contributions from the community. To participate effectively:

1. **Check Existing Reports:** Before creating a new issue, ensure the problem has not been reported or addressed.
2. **Provide Detailed Information:** When reporting an issue, include your platform, compiler version, steps to reproduce, and any error messages.
3. **Follow Contribution Guidelines:** See the [CONTRIBUTING.md](https://github.com/google/googletest/blob/main/CONTRIBUTING.md) in the repository for standards, patch submission details, and coding style.
4. **Ask Questions on Community Channels:** Use GitHub Discussions or Stack Overflow for usage questions or troubleshooting help.

## Additional Help Resources

- **FAQ Section:** The official FAQ provides answers to common problems and questions about building, running, and debugging tests.
- **Troubleshooting Guides:** The documentation includes dedicated pages for [Common Installation Problems](https://google.github.io/googletest/installation-guide/common-problems.html) and [Build System Configuration Pitfalls](https://google.github.io/googletest/installation-guide/build-system-issues.html) which can save you time.
- **Sample Tests:** Reviewing sample test files can be very instructive; see the [Samples](https://google.github.io/googletest/samples.html) page.

## Tips for Effective Use of Help Channels

- Search the existing documentation and forums **before** posting questions.
- Use clear, descriptive titles and tags.
- Provide code snippets, configuration files, and error outputs when relevant.
- Respect community guidelines and be patient — contributors volunteer their time.

By leveraging the official documentation alongside community resources, you can continuously improve your testing skills and quickly overcome obstacles with GoogleTest and GoogleMock.
