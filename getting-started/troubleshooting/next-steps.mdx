# Where to Go Next

Welcome to the next stage of your GoogleTest learning journey. This page guides you through a curated set of resources designed to deepen your expertise, expand your testing skills, and unlock advanced features of GoogleTest and GoogleMock.

## Recommended Documentation to Explore

### 1. GoogleTest Primer
- **Purpose:** Start with this guide if you are new to GoogleTest or want a refresher on writing basic tests.
- **Focus:** Writing simple test macros, test discovery, and running your first test.
- **Access:** [GoogleTest Primer](/getting-started/first-test-run/create-first-test)

### 2. Advanced Topics
- **Purpose:** Once comfortable with the basics, explore in-depth details about advanced assertion use, fixtures, parameterized tests, and complex testing strategies.
- **Access:** [Advanced Topics](/guides/core-workflows/assertions-and-matchers)

### 3. Mocking for Dummies & Mocking Cookbook
- **Purpose:** Learn how to create, configure, and use mock objects effectively with GoogleMock.
- **Highlights:** Defining mock classes, setting expectations, handling complex argument matchers, and best practices for maintaining clean mock-based tests.
- **Access:** [Mocking for Dummies](/docs/gmock_for_dummies.html), [Mocking Cookbook](/docs/gmock_cook_book.html)

### 4. Integration and Build Systems
- **Purpose:** Understand how to integrate GoogleTest and GoogleMock into your build environment using CMake or Bazel, including troubleshooting integration pitfalls.
- **Access:** [Integration with Build Systems](/overview/use-cases-integration/integration-with-build-systems)

### 5. Troubleshooting and FAQ
- **Purpose:** Prepare for or resolve common issues during setup and test execution.
- **Access:** [Troubleshooting Common Setup Issues](/getting-started/troubleshooting/common-setup-issues), [GoogleTest FAQ](/docs/faq.html)

## Suggested Next Steps

1. **Practice Writing Tests:** Use the Primer and First Test Run guides to create and run your tests.
2. **Explore Mocks:** Dive into mocking documentation to write effective interaction tests.
3. **Understand Assertions and Matchers:** Master advanced assertions and matcher usage for precise verification.
4. **Configure Your Build:** Set up integration with your preferred build system for streamlined testing.
5. **Leverage Troubleshooting:** If you encounter issues, consult the troubleshooting guides promptly to minimize delays.

## Getting Support and Community
