# Getting Help & Community Resources

Empowering your journey with GoogleTest and GoogleMock means knowing where to turn for answers, connecting with peers, and contributing back to the project. This page guides you through the rich ecosystem of community Q&A, discussion forums, official documentation, and best practices for reporting issues and submitting improvements.

---

## 1. Community Q&A and Discussion Forums

GoogleTest and GoogleMock boast an active user community eager to help and share knowledge. Engaging with these channels connects you to experienced users and maintainers.

### Recommended Platforms

- **Stack Overflow**
  - Search and ask questions tagged with [`googletest`](https://stackoverflow.com/questions/tagged/googletest) or [`googlemock`](https://stackoverflow.com/questions/tagged/googlemock).
  - Tip: Before posting, search for similar questions to avoid duplicates.

- **GitHub Issues**
  - The issue tracker for the [googletest GitHub repo](https://github.com/google/googletest/issues) is a primary place for reporting bugs and feature requests.
  - Use clear, descriptive titles and provide reproducible examples to get effective support.

### Best Practices for Asking Questions

- Provide your environment details: OS, compiler version, GoogleTest/gMock version.
- Include minimal, reproducible examples of your test code.
- Clearly describe what you expect to happen, and what actually occurs.

<Tip>
To increase your chances of receiving fast, helpful answers, make sure your question is precise and shows your troubleshooting steps.
</Tip>

## 3. How to Report Issues or Contribute Fixes

Your contributions make GoogleTest and GoogleMock better for everyone. Follow these guidelines to report problems or submit patches effectively.

### Reporting Issues

1. **Verify the problem:**
   - Ensure your bug is reproducible using the latest release or main branch build.
   - Isolate the issue with minimal test code.
2. **Search existing issues:**
   - Check if someone else has reported a similar problem.
3. **Create a new GitHub issue:**
   - Provide a descriptive title.
   - Include environment details: OS, compiler and version, GoogleTest/gMock version.
   - Attach minimal reproducible code snippets.
   - Describe expected and actual results.

### Submitting Fixes or Enhancements

1. **Fork the repository** and create a feature branch.
2. **Follow the existing code style and naming conventions.**
3. **Add tests** to cover the changes.
4. **Run all tests** locally to ensure nothing breaks.
5. **Submit a Pull Request (PR)** with clear description and motivation.

<Note>
Review the contributing guidelines in the repository before submitting your PR to adhere to project standards.
</Note>
