## Version History

### 1.17.0 (Released May 2024)
- Upgraded minimum C++ standard support to C++17 to leverage modern language features and improve cross-platform compatibility.
- GoogleTest and GoogleMock projects merged into a single repository for streamlined maintenance and release management.
- Documentation has transitioned to a dedicated GitHub Pages site for enhanced navigation and searchability: https://google.github.io/googletest/
- Introduced default dependency planning for Abseil (absl) in upcoming versions.

### 1.16.0 and earlier
*(This history is abbreviated here; please refer to official release notes on GitHub or product documentation for detailed prior entries.)*

- Added death test support and enhancements to assertion macros for crash and error condition validation.
- Introduced value-parameterized and type-parameterized tests for efficient broad-spectrum testing.
- Improved mocking behaviors with strictness wrappers: NiceMock, NaggyMock, and StrictMock.
- Enhanced platform support and embedded system integration (e.g., ESP32, NRF52).
- Streamlined build system integration with CMake and Bazel, including recommended setup workflows and troubleshooting guides.

---

## Highlights of Key Past Releases

- **Test Discovery and Execution Improvements:** Automatic global registration of tests via macros; lifecycle hooks for setup and teardown at suite and test levels.
- **Assertions Enrichment:** Expanded set of rich assertion macros including predicate assertions and floating-point comparisons.
- **Death Tests:** Support for tests that verify process termination with controllable matchers and execution styles.
- **Mocking API Evolution:** Control over mock object behavior and expectations with flexible ON_CALL and EXPECT_CALL macros.
- **Platform-Targeted Enhancements:** Better handling for platform-specific initialization, threading, and portability flags.
