# Latest Release Details

Explore the key updates, improvements, and essential changes introduced in the most recent release of GoogleTest. This document dives into new features, bug fixes, and any breaking changes you need to know to smoothly upgrade and leverage the latest enhancements.

- **Release Version:** 1.17.0
- **Release Date:** Available as of [GoogleTest GitHub Release v1.17.0](https://github.com/google/googletest/releases/tag/v1.17.0)

GoogleTest 1.17.x requires a compiler and environment supporting at least the C++17 standard. This update aligns with the evolving C++ landscape and ensures modern language features are consistently available.

## 3. New Features and Enhancements

### 3.1 C++17 Standard Requirement

To reflect the contemporary C++ development environment, GoogleTest 1.17.x explicitly requires C++17. This means:

- Your compiler and build settings must target C++17 or newer.
- Legacy C++11 or C++14 environments are no longer supported on this branch.

This ensures the framework can utilize new C++ features to improve testing infrastructure robustness and maintainability.

### 3.2 Preparations for Abseil Integration

GoogleTest is preparing to adopt a dependency on [Abseil](https://github.com/abseil/abseil-cpp), a collection of C++ libraries developed by Google. While this release does not yet pull in Abseil by default, it introduces build options to enable experimental use:

- A `GTEST_HAS_ABSL` build flag allows advanced users to compile with Abseil support if the dependencies are available manually.
- This paves the way for future performance and feature enhancements leveraging Abseil's utilities.

### 3.3 Build and Platform Improvements

- Improved CMake support aligns with version 3.16 or higher, offering a flexible and modern cross-platform build experience.
- By default, C++ extensions are disabled on supported platforms to promote portability and conformance.
- GoogleMock continues to build atop GoogleTest, with options to build only the testing framework if desired.

## 4. Bug Fixes and Stability Enhancements

- Fixed multi-threaded test execution behavior on select platforms, enhancing thread safety and deterministic outcomes.
- Corrected issues around environment detection macros ensuring consistent behavior across Windows, Linux, macOS, and embedded platforms.
- Resolved linker and runtime errors that appeared when mixing static and shared library usage in Windows Visual Studio projects.
- Enhanced error logging for assertion and test failures providing clearer diagnostics.

These fixes contribute to a more stable and reliable testing framework for all users.

## 5. Breaking Changes and Upgrade Guidance

### 5.1 C++17 Requirement

Your build environment must support C++17 and be configured accordingly. Upgrade your compiler and build flags usually via:

```cmake
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
```

Failing to do so will result in compilation errors.

### 5.2 Removal of Legacy Support

Any customized configurations or extensions relying on earlier C++ standards or deprecated macros may require refactoring.

### 5.3 Linking and Runtime Flags

If linking against shared GoogleTest libraries on Windows, ensure to define:

```
-DGTEST_CREATE_SHARED_LIBRARY=1
-DGTEST_LINKED_AS_SHARED_LIBRARY=1
```

in your compiler flags to maintain compatibility.

### 5.4 Test Macro Name Clashes

If you previously worked around macro name clashes using `-DGTEST_DONT_DEFINE_FOO=1`, verify these remain compatible and adjust your code to use renamed macros if needed.

## 6. How to Upgrade Smoothly

Follow this user-oriented workflow:

1. **Verify Your Toolchain:** Confirm your compiler supports C++17 and update your build configuration.
2. **Pull the Latest Version:** Clone or update your GoogleTest source to v1.17.0.
3. **Adjust Build Scripts:** Ensure your CMake scripts and build environment account for the new requirements and flags.
4. **Run Existing Tests:** Execute your test suites unchanged to detect any integration issues.
5. **Use `gtest_main` for Convenience:** Link with the `gtest_main` library to avoid custom main() implementations.
6. **Enable Abseil Support (Optional):** If desired, build with `-DGTEST_HAS_ABSL=1` and provide Abseil dependencies.

Consider consulting the [Upgrade Guide](https://github.com/google/googletest/blob/main/docs/changelog/breaking-changes-upgrade/upgrade-guide.md) for detailed migration steps.

### Example: Typical CMake Build Setup for v1.17.0

```bash
git clone https://github.com/google/googletest.git -b v1.17.0
cd googletest
mkdir build && cd build
cmake ..
make
sudo make install
```

To build only GoogleTest (no GoogleMock):

```bash
cmake .. -DBUILD_GMOCK=OFF
```

Ensure your compiler supports C++17 and your project configures CMake accordingly.
