# Latest Release Notes

This document provides a detailed breakdown of the most recent release of GoogleTest and GoogleMock, highlighting key features, resolved issues, minor improvements, and any upgrade instructions. It is designed to help users quickly understand what has changed, enabling them to benefit from the latest advancements and maintain compatibility.

---

## 1. Executive Summary
- **Version**: 1.17.0
- **Release Date**: [Refer to official GitHub release page](https://github.com/google/googletest/releases/tag/v1.17.0)
- **Key Highlights**:
  - Minimum C++17 standard requirement enforced
  - Enhanced test discovery, execution, and mocking improvements
  - Ongoing modernization with planned Abseil dependency

This release builds on the foundation of earlier versions by tightening the language requirements and improving integration capabilities, helping to ensure robust, modern C++ testing workflows.

## 2. New Features and Improvements

### 2.1 Language and Platform Requirements
- The 1.17.x branch now requires C++17 or newer, which enables improved language features and more efficient test implementations.
- This change ensures GoogleTest stays current with modern C++ standards, facilitating better performance and compatibility on supported compilers and platforms.

### 2.2 Test Discovery and Execution Enhancements
- Improved auto-discovery of tests continues to simplify running tests without explicit registration.
- Enhancements to test suite lifecycle management yield more reliable fixture setup and teardown.

### 2.3 Mocking Support Updates
- GoogleMock methods continue to be unified under the `MOCK_METHOD` macro.
- Improvements in strictness modes and call expectations promote clearer and more maintainable mock definitions.

### 2.4 Build and Integration
- Better integration with build systems like CMake and Bazel is maintained, with community-supported CMake scripts included.
- Updated pkg-config files deliver more consistent linking and compilation configuration.

### 2.5 Documentation and Ecosystem
- Documentation has been migrated to GitHub Pages with richer navigation.
- Links to community tools, runners, and VS Code extensions have been updated.

## 3. Bug Fixes and Minor Enhancements
- Fixed several issues related to thread safety in death tests.
- Resolved corner cases in parameterized test instantiation.
- Addressed flaky behavior in strictness mode triggers.
- Various improvements in matcher diagnostics and failure message clarity.

## 4. Upgrade and Compatibility Notes

### 4.1 C++ Standard Enforcement
- Users upgrading must ensure their projects compile with at least C++17.
- This requirement affects compiler flags and may impact projects using older standards.

### 4.2 Migration of MOCK_METHOD Usage
- Transition from legacy `MOCK_METHODn` macros to the unified `MOCK_METHOD` macro is recommended for new tests.
- Existing mocks continue to function but updating is advised for maintainability.

### 4.3 Build Configuration
- Review and adjust build system configurations, especially if using CMake, to align with updated scripts and dependencies.

### 4.4 Future Planned Changes
- A future dependency on the Abseil library is announced, which will aid in modernizing internals.
- Users should anticipate this change and plan gradual migration strategies accordingly.

## 5. How to Get the Release

### 5.1 Download
- Obtain the latest source from [GitHub Releases](https://github.com/google/googletest/releases/tag/v1.17.0) or by checking out the `main` branch.

- Refer to the official [GoogleTest User's Guide](https://google.github.io/googletest/) for detailed steps.

### 5.3 Validation
- After installation, validate with a simple test to ensure environment and dependencies are correct.

## Appendix: Example Upgrade Steps

### Step 1: Update Compiler Flags
```bash
# For GCC or Clang, add:
-std=c++17
```

### Step 2: Upgrade Mock Definitions
```cpp
class MockFoo {
  // Old way
  // MOCK_METHOD1(Bar, void(int));
  
  // New recommended way
  MOCK_METHOD(void, Bar, (int), (override));
};
```

### Step 3: Rebuild and Validate
```bash
cmake -S . -B build && cmake --build build && build/tests/test_binary
```
