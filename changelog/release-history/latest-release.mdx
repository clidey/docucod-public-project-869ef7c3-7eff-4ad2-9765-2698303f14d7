Stay up to date with the most recent release of GoogleTest, the leading C++ testing framework from Google. ## Release 1.17.0 Highlights

The newest release of GoogleTest, version **1.17.0**, comes with important modernizations and refinements designed to improve both compatibility and development efficiency.

### Key Features and Improvements

- **C++17 Minimum Standard Enforced**
  - The 1.17.x branch now requires at least C++17 to build and run.
  - This change leverages modern C++ features to offer cleaner, safer, and potentially more performant code.

- **Unified GoogleTest and GoogleMock**
  - Continued support for seamless integration of GoogleTest and GoogleMock, now maintained and released as a single project for consistency and ease of use.

- **Documentation Access via GitHub Pages**
  - Official documentation is hosted and continually updated on GitHub Pages at [https://google.github.io/googletest/](https://google.github.io/googletest/).
  - Users are encouraged to browse this resource to get the latest guides, API references, and examples.

- **Improved Continuous Integration**
  - GoogleTest's build and test processes are validated via Google's internal CI systems, ensuring release stability.

- **Upcoming Dependency on Abseil**
  - In future releases, GoogleTest plans to depend on Abseil (Google's common libraries), reflecting a commitment to improved modularity and modern infrastructure.

### Immediate Benefits for Your Development Workflow

- **Modern Compiler Support**
  - Upgrade your toolchain to C++17 or newer to benefit from enhanced language features and library support.

- **Streamlined Testing Experience**
  - Take advantage of the combined GoogleTest and GoogleMock repository for easier setup and maintenance.

- **Access Dynamic and Updated Documentation**
  - Quickly find tutorials, reference material, and best practices online without digging through source code or outdated files.

- **Community and Ecosystem Support**
  - Benefit from integration with popular test runners, IDE extensions, and improved tooling as the project evolves.

## How to Get Started with Version 1.17.0

### Cloning and Building

Follow the steps below to get started with the latest version:

```bash
# Clone the repository at the 1.17.0 tag
git clone https://github.com/google/googletest.git -b v1.17.0
cd googletest

# Create a build directory
mkdir build && cd build

# Generate the build files with CMake
cmake ..

# Build the project
make

# (Optional) Install to system directories
sudo make install
```

> **Tip:** If you want to build only GoogleTest without GoogleMock, add `-DBUILD_GMOCK=OFF` to the `cmake` command.

## Upgrade and Compatibility Considerations

- **C++ Standard Compliance**
  - Ensure your build environment supports C++17 as earlier standards are no longer supported.

- **Build System Changes**
  - If upgrading from earlier versions, adapt your `CMakeLists.txt` or build scripts to accommodate the enforced standard and any related dependency changes.

- **API Stability**
  - There are no breaking API changes in this release, but reviewing the [Breaking Changes & Migration Notes](https://github.com/google/googletest/releases/tag/v1.17.0) is recommended.
