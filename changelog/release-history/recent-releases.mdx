# What's New: Recent Releases

Explore the latest enhancements and fixes in GoogleTest with an in-depth focus on major and minor releases. This page guides you through recent updates that directly impact your testing workflows, helping you decide when and how to upgrade to the newest versions.

---

## Recent Major and Minor Releases Overview

Recent releases of GoogleTest continue to build on a solid foundation of reliable, feature-rich C++ testing capabilities. Each update introduces practical improvements, new features, performance optimizations, and key fixes designed to enhance your testing experience.

### Version 1.17.0 (Latest Release)

Released recently, version 1.17.0 brings important modernizations and compatibility enhancements:

- **Mandatory C++17 Standard**: This release requires building with at least C++17, ensuring access to modern C++ features, improved compiler diagnostics, and better standard library support.

- **Streamlined Integration**: Incorporation of Abseil as a direct dependency is planned soon, signaling an improved modular and extensible architecture for upcoming versions.

- **Documentation Improvements**: The documentation site has been migrated and enriched on GitHub Pages, making it easier to browse up-to-date guides, API references, and samples.

- **Robust Platform Compatibility**: Continuous efforts ensure broad compatibility with major compilers, IDEs, and build systems while aligning with Google's foundational C++ support policy.

This release focuses on maintaining compatibility, scaling with modern toolchains, and preparing for deeper integrations with Google's utility libraries.

> **Note:** If your projects currently use an older C++ standard, upgrading to 1.17.0 will require compiling with at least C++17. Consult the [System Prerequisites](../getting-started/setup-overview/system-prerequisites) documentation for detailed environment requirements.

### Highlights from Prior Releases

While the latest release brings key adjustments, incremental updates over recent versions have included:

- **Enhanced Parameterized and Typed Tests**: Improved APIs allowing more flexible parameter generation, custom naming, and instantiation strategies.

- **Improved Mocking API Flexibility**: Additional matchers, actions, and granular mock object configuration options to control uninteresting calls and call expectations.

- **Expanded Platform and Integration Support**: Better compatibility with emerging build systems such as Bazel, refined CMake integration patterns, and IDE support.

- **Scalability and Performance Optimizations**: Facilities for parallel test execution, sharding, and shared resource management to accelerate large test suites.

- **Bug Fixes and Stability Enhancements**: Numerous fixes addressing corner cases in death tests, assertion diagnostics, and multi-threaded execution.

Refer to the [Version Timeline & Release Notes](./version-timeline) page for a comprehensive chronological history and extensive details.

## How This Affects Your Testing Workflow

### Choosing When to Upgrade

- Upgrading to the latest releases like 1.17.0 ensures you leverage improved diagnostics, modern C++ support, and performance enhancements.
- Consider your project's compiler environment and C++ standard compatibility as prerequisites before migrating.
- Check for any deprecated APIs or breaking changes detailed in the breaking changes documentation before upgrading.

### Leveraging New Features

- Adopt updated parameterized testing APIs to write more reusable and maintainable tests covering diverse inputs and types.
- Utilize enhanced mocking strictness modes and configuration to balance test rigor with log verbosity effectively.
- Benefit from speed improvements by leveraging test sharding and parallel execution techniques documented in the scaling guide.

### Best Practices

- Always verify your test environment against the [System Prerequisites](../getting-started/setup-overview/system-prerequisites).
- Review integration patterns for your build system ([CMake](../guides/integration_and_effectiveness/build_system_integration), [Bazel]) to align with changes.
- Follow recommended migration guides when upgrading across major versions to prevent disruptions.
