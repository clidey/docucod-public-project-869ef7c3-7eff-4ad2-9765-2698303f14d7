# Detailed Release Notes

This page provides an aggregated view of granular release notes for each published version or patch of GoogleTest and GoogleMock. Here, you will find summaries of improvements, fixes, and changes made to the testing framework, enabling you to make informed upgrade decisions. Where available, links to external GitHub release pages offer detailed change logs and commit histories.

## Release Notes Structure and Use

This documentation organizes release notes by version, with the newest releases at the top to quickly surface the latest enhancements.

Each version entry includes:

- **Version Number & Release Date:** Identifies the release for effective change management.
- **Highlights:** Key features, improvements, and fixes that impact user workflows.
- **Breaking Changes:** Clear callouts for any changes requiring user migration or code adaptation.
- **Deprecations:** Features marked for removal in future versions, with recommendations.
- **Security Updates:** Any patches relevant to security vulnerabilities.
- **Links:** Direct navigation to the official GitHub release page with complete commit and issue tracking details.

Users should consult these notes before updating to understand potential impacts and benefits.

## Recent Major Releases

### Version 1.17.0 (Released September 2022)

- **Highlights:**
  - GoogleTest now requires **C++17 or newer** for all supported platforms, aligning with modern C++ standards.
  - This release merges GoogleMock and GoogleTest into a unified repository and release cycle for tighter integration.
  - Planned introduction of a dependency on **Abseil**, Google's common C++ library, is signalled for future versions.

- **Breaking Changes:**
  - Dropped support for compilers and platforms not meeting the C++17 requirement.
  - Users must update their build environments accordingly.

- **Security Updates:** None reported.

- **Reference:** Full details available at [GoogleTest GitHub Release v1.17.0](https://github.com/google/googletest/releases/tag/v1.17.0).

## Upgrade and Compatibility Notes

- Ensure that your build environment supports at least **C++17**, as all current releases mandate this requirement.
- Before upgrading, verify your use of deprecated features outlined in previous deprecation notices to avoid build failures.
- Review any changes to GoogleMock APIs or mocking behavior that may affect test expectations and verifications.
