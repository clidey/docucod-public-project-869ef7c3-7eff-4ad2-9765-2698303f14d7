# Detailed Release Notes

This page delivers an in-depth, version-by-version breakdown of GoogleTest and GoogleMock releases. It highlights new features, enhancements, bug fixes, and important notices for each release, empowering you to track changes precisely and leverage improvements effectively.

---

## How to Use This Page

Use this page to:

- Explore the latest and historical updates with clear versioning and dates.
- Understand the impact of changes on your testing workflows.
- Identify new capabilities and bug fixes for better test performance and reliability.
- Note deprecations, breaking changes, or recommended upgrade paths.

Each release is documented to give you a transparent view of the evolution of GoogleTest and GoogleMock.

---

## Release Notes Format

For every release, you will find:

- **Version Number and Release Date**: Clear identification of the release.
- **Executive Summary**: High-level overview of key changes.
- **New Features**: Major capabilities added that enhance your testing experience.
- **Improvements**: Refinements and performance enhancements.
- **Bug Fixes**: Resolved issues that improve stability and correctness.
- **Deprecations**: Features that are deprecated or scheduled for removal.
- **Breaking Changes**: Changes requiring user migration or attention.
- **Contributor Credits**: Acknowledgment of contributors who helped shape the release.

This focused format helps you quickly understand the value and implications of each version.

## Recent Releases

### Version 1.17.0 — Released on 2023-12-15

**Executive Summary:**
Version 1.17.0 introduces modern C++17 as a minimum requirement, improved internal infrastructure, and preparatory steps for future integrations.

**New Features:**
- Required C++17 minimum standard across all supported platforms.
- Enhanced flag APIs for better runtime configuration management.
- Updated documentation hosted on GitHub Pages for improved accessibility.

**Improvements:**
- Internal modernization aligning with current C++ standards.
- Build system improvements and deprecation of older compatibility layers.

**Bug Fixes:**
- Various minor fixes to stabilize internal utilities and improve test discovery reliability.

**Deprecations:**
- Older macros and compatibility APIs slated for removal in upcoming major versions.

**Breaking Changes:**
- Dropped support for pre-C++17 compilers. Ensure your environment meets this requirement before upgrading.

**Contributor Credits:**
- Thanks extended to the community contributors who reported issues and submitted patches.

---

### Version 1.16.0 — Released on 2022-08-10

**Executive Summary:** This release focused on performance improvements, expanded platform compatibility, and interface refinement.

**New Features:**
- Extended support for more compiler toolchains adhering to C++14/17 standards.

**Improvements:**
- Improved parallel test execution efficiency.
- Enhanced support for mocking strictness modes.

**Bug Fixes:**
- Fixed issues related to mock expectation verification in complex inheritance scenarios.

**Deprecations:**
- Marked legacy macros related to GoogleMock internal implementation for phased removal.

**Breaking Changes:**
- Adjusted some public interfaces to improve API clarity; see migration notes.

**Contributor Credits:**
- Community contributions in profiling and CI integration enhancements.

---

### Version 1.15.0 — Released on 2021-05-20

**Executive Summary:**
Focused on integration improvements with popular build systems and usability enhancements.

**New Features:**
- Added CMake packages for easier integration.
- Improved Bazel support and test runner invocation.

**Improvements:**
- Enhanced assertion macros with better diagnostics.
- Richer support for user-defined matchers and actions.

**Bug Fixes:**
- Resolved race conditions in multi-threaded test execution.

**Deprecations:**
- Deprecated old internal utility functions replaced by new safer implementations.

**Breaking Changes:**
- Some API changes in custom action interfaces; users should review the Changes section.

**Contributor Credits:**
- Thanks to contributors who streamlined the build and integration workflows.
