## Release Types

- **Major releases** (e.g., 1.17.0) introduce new features and can sometimes include breaking changes requiring migration steps.
- **Minor releases** (e.g., 1.16.0) usually bring improvements, new APIs, and backward-compatible enhancements.
- **Patch releases** focus on fixes, security updates, and minor performance improvements without impacting compatibility.

---

## Versioning Policy

GoogleTest follows Semantic Versioning (SemVer), where version numbers are presented as `MAJOR.MINOR.PATCH`:

- Increment **MAJOR** when making incompatible API changes.
- Increment **MINOR** for adding functionality in a backward-compatible manner.
- Increment **PATCH** for backward-compatible bug fixes.
