# All Release Notes

Explore the complete chronological history of GoogleTest and GoogleMock releases in one centralized location. This page serves users who want to understand the evolution of features, improvements, bug fixes, and known issues across all versions.

Each version entry provides a clear snapshot of what changed, helping you make informed decisions about upgrades and compatibility.

---

## How to Use the All Release Notes Page

The page is structured to give you an at-a-glance understanding of each release, including:

- **Version and Release Date**: Know exactly when each version became available.
- **New Features**: Discover new capabilities introduced to enhance your testing and mocking.
- **Improvements**: Learn about refinements and performance boosts made to existing features.
- **Bug Fixes**: Find corrections that improve stability and reliability.
- **Known Issues**: Be aware of any remaining or emerging problems to better plan your testing environment.

Use this section to identify when specific functionality appeared or evolved, or to review changes impacting migration or testing strategies.

## Best Practices for Upgrading

- **Read the Latest Release Notes First**: Focus first on the most recent changes to address immediate upgrade considerations.
- **Check Known Issues**: Anticipate and plan to mitigate known problems reported for your target version.
- **Review Feature Additions**: Understand new capabilities that might enable simpler or more robust test implementations.

## Troubleshooting Common Problems

If your tests start to fail or behave unexpectedly after upgrading, consider:

- Verifying that the mock expectations and default actions accord with the newer mocking APIs.
- Adjusting your use of the NiceMock, NaggyMock, or StrictMock wrappers to align with updated behaviors.
- Running tests with increased verbosity (`--gmock_verbose=info`) to obtain detailed call and expectation traces.
- Consulting the [Troubleshooting Setup Problems](../getting-started/troubleshooting-faq/setup-troubleshooting) page.
