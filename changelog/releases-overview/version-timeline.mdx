# Version Timeline

The Version Timeline page presents a comprehensive, chronological list of all released versions of GoogleTest and GoogleMock. It provides clear release dates for each version, enabling users to track the progression and pace of development. This historical overview allows users to identify specific versions relevant to their projects and easily navigate to detailed release notes for each version.

---

## Purpose and User Benefits

If you maintain C++ projects using GoogleTest or GoogleMock, understanding the version history helps you:

- **Track updates** to the framework over time, including new features, bug fixes, and deprecations.
- **Locate versions** compatible with your environment or with specific capabilities you need.
- **Plan upgrades** by reviewing the timeline of releases and adopting the latest stable versions.
- **Understand release frequency** to gauge how actively the project evolves.

This page acts as a centralized timeline and entry point for release details, allowing you to visualize GoogleTest's growth and maturity.

---

## Page Structure and Navigation

The timeline lists versions in **reverse chronological order**, placing the most recent releases at the top for easy access. Each entry includes:

- **Version number** following semantic versioning conventions (e.g., `1.17.0`).
- **Release date** formatted with exact day, month, and year to contextualize updates.
- **Clickable links** directing users to the full [Release Notes](https://google.github.io/googletest/changelog/releases-overview/release-notes) page for in-depth details on features, fixes, deprecations, and migration guidance.

This logical layout helps users quickly locate the version that interests them and jump straight to specific change logs.

## Example Entry Format

Each version entry typically appears as:

- `Version X.Y.Z` (Release Date: YYYY-MM-DD)

For example:

- **Version 1.17.0** (Released: 2023-10-21) — See detailed notes [here](https://github.com/google/googletest/releases/tag/v1.17.0).
