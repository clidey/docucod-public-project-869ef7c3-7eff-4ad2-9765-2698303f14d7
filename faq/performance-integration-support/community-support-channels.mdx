# Community & Support Resources

The **Community & Support Resources** page serves as your central gateway to external tools, community contributions, support channels, and participation opportunities related to GoogleTest and GoogleMock. Here you will find curated pointers to user guides, forums, issue trackers, and community-created documentation that can accelerate your adoption, troubleshooting, and mastery of GoogleTest.

---

## Frequently Asked Questions (FAQs)

### Where can I find community-contributed GoogleTest documentation?

GoogleTest benefits from documentation created by its vibrant community. For example, the repository hosts interesting reads like [Googlemock Insights](https://github.com/ElectricRCAircraftGuy/eRCaGuy_dotfiles/blob/master/googletest/insights.md) contributed by community members. These documents often offer unique viewpoints, practical tips, and tutorials beyond the official docs.

### How can I get official guidance and user manuals for GoogleTest?

We recommend starting with the comprehensive [GoogleTest User's Guide](https://google.github.io/googletest/), which provides tutorials, primers, and reference material. For a rapid start, the [GoogleTest Primer](https://google.github.io/googletest/primer.html) is ideal for new users seeking hands-on examples.

### Are there external tools that integrate with GoogleTest?

Yes, the community has built many tools enhancing your GoogleTest experience, such as:

- **GTest Runner:** A Qt5-based automated test-runner and GUI for Windows and Linux.
- **GoogleTest UI:** A C# test runner featuring a progress bar and clickable failure logs.
- **GTest TAP Listener:** Provides TAP protocol test output for compatible runners.
- **gtest-parallel:** Accelerates test execution by running tests in parallel.
- **VS Code Extensions:** Including [GoogleTest Adapter](https://marketplace.visualstudio.com/items?itemName=DavidSchuldenfrei.gtest-adapter) and [C++ TestMate](https://github.com/matepek/vscode-catch2-test-adapter) to run and debug tests within the editor.
- **Cornichon:** A Gherkin DSL parser that generates GoogleTest stub code.

Using such tools can streamline test development, debugging, and reporting.

### How do I get help when I encounter issues?

Several avenues are available:

- **Issue Tracker:** Report bugs or request features at the [GoogleTest GitHub Issues](https://github.com/google/googletest/issues).
- **Mailing List and Forums:** Join the `googletestframework@googlegroups.com` mailing list to ask questions, discuss features, and share knowledge.
- **Community Forums:** Explore discussions on platforms like Stack Overflow tagged with `googletest`.

### How can I contribute to GoogleTest or get involved in the community?

Joining the community and contributing is encouraged and welcome:

- Read the [`CONTRIBUTING.md`](https://github.com/google/googletest/blob/main/CONTRIBUTING.md) guide to understand contribution workflow, coding standards, and submission guidelines.
- Participate in discussions in GitHub issues and pull requests.
- Share your own documentation, tools, or examples to help fellow users.

Your engagement helps improve the project and support ecosystem.

## Support Channels

- File bug reports, feature requests, or ask questions via the [GitHub Issues page](https://github.com/google/googletest/issues).

- Join the GoogleTest mailing list: `googletestframework@googlegroups.com` for direct interaction with the maintainers and users.

- Contributions to docs as pull requests on the GitHub repository are highly valued.

## Participating and Contributing

Engaging in the GoogleTest community is straightforward:

1. **Read the Contribution Guidelines:** The [`CONTRIBUTING.md`](https://github.com/google/googletest/blob/main/CONTRIBUTING.md) file explains how to submit code, report issues, and what coding style to follow.

2. **Fork and Clone the Repo:** Work on your enhancements or fixes locally.

3. **Create Pull Requests:** Propose your changes for review.

4. **Report Issues or Suggest Features:** Opening well-documented issues helps prioritize work.

5. **Share Your Tools and Documentation:** Help others by contributing external resources or improvements.

Your contributions, whether code, documentation, or feedback, foster the growth and robustness of GoogleTest.
