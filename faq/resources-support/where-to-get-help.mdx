# Where to Get Help and Report Issues

This page describes the preferred ways to get help with GoogleTest, report bugs, and request new features. Whether you are a new user or an experienced developer, knowing where and how to seek assistance or contribute effectively will streamline your experience and help improve GoogleTest for everyone.

---

## Getting Help

When you encounter issues or have questions about using GoogleTest, several resources are available to guide and support you:

### 1. Official Documentation

GoogleTest provides comprehensive documentation that covers everything from basic usage to advanced techniques:

- **[GoogleTest Primer](primer.md)** – Start here if you are new to GoogleTest; it teaches essential concepts and usage.
- **[GoogleTest Advanced](advanced.md)** – For advanced features and detailed guidance.
- **[Assertions Reference](reference/assertions.md)** – Detailed explanations of assertion macros.
- **[Mocking Reference](reference/mocking.md)** – Instructions and reference for mocking facilities.
- **[gMock For Dummies](gmock_for_dummies.md)** and **[gMock Cookbook](gmock_cook_book.md)** – Practical guides on using mocks effectively.
- **[GoogleTest FAQ](faq.md)** and **[gMock FAQ](gmock_faq.md)** – Frequently asked questions and tips.

Consult these documents regularly—they cover most common scenarios and user questions.

### 2. Source Code Samples

GoogleTest includes sample tests that demonstrate correct usage patterns and how to implement your own tests:

- Review the provided black-box tests and samples in the repository to see real examples.

### 3. Community Forums and Discussions

GoogleTest's open-source community is active and supportive.

- **GitHub Issues:** Participate in discussions, search for existing issues, or raise new ones at the [googletest GitHub repository](https://github.com/google/googletest).
- **Stack Overflow:** Tag your questions with `googletest` for community help.
- **Mailing Lists and Chat:** Explore GoogleTest community channels for interactive help.

### 4. Contribution Guidelines

If you are interested in contributing fixes, improvements, or new features, please read the contribution guidelines available in the GitHub repository's README and `CONTRIBUTING.md` files.

- Follow best practices in coding, testing, and documenting your changes.
- Ensure that your contributions are properly tested.

---

## Reporting Bugs

To report a bug effectively and help maintainers understand your issue, please:

- Describe the problem clearly and concisely.
- Include the version of GoogleTest you are using.
- Provide a minimal reproducible example.
- Attach logs, error messages, and system environment details when pertinent.
- Check existing issues to avoid duplicates.

Bug reports should be submitted through the [GitHub Issues](https://github.com/google/googletest/issues) tracker.

---

## Requesting Features

New features or enhancements are welcome and can be requested by:

- Detailing your desired feature or improvement.
- Explaining the use case and benefits.
- Suggesting possible approaches if you have ideas.

Feature requests are best submitted as issues on the [GitHub repository](https://github.com/google/googletest/issues). Consider discussing significant changes beforehand via proposals or discussions as appropriate.
