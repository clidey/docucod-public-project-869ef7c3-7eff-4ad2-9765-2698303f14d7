# Where to Get Help & Community Channels

This page guides you to the official discussion forums, GitHub issues, community-created guides, and other recognized channels where you can ask questions, report bugs, or find support for complex or edge-case challenges related to GoogleTest.

---

## Official Support Channels

### GitHub Issues

The primary channel for reporting bugs, requesting features, or raising detailed issues with GoogleTest is the GitHub repository's Issues tracker:

- Repository: [https://github.com/google/googletest](https://github.com/google/googletest)
- Issues page: [https://github.com/google/googletest/issues](https://github.com/google/googletest/issues)

**Best Practices When Opening Issues:**

- Search existing issues first to avoid duplicates.
- Provide a clear, minimal, and reproducible example.
- Specify your environment, compiler, and GoogleTest version.
- Include error messages or unexpected behavior details.

### Mailing Lists and Forums

While GoogleTest does not maintain an official mailing list, you may find support through popular developer platforms like Stack Overflow by tagging your questions with `googletest`.

- [Stack Overflow GoogleTest Questions](https://stackoverflow.com/questions/tagged/googletest)

Community members and maintainer contributors actively answer questions here.


## Community Resources

### Community-Created Guides and Tutorials

Several developers have created detailed guides, tutorials, and example test repositories, which can supplement the official documentation:

- [GoogleTest Primer](https://github.com/google/googletest/blob/main/docs/primer.md) — Official beginner-friendly tutorial.
- [gMock for Dummies](https://github.com/google/googletest/blob/main/googlemock/docs/gmock_for_dummies.md) — Guides mocking with GoogleMock.
- Various community blog posts and video tutorials available via web search.

### User-Contributed Questions & Answers

Many experienced GoogleTest users share their solutions to common problems and advanced usage scenarios on forums such as:

- Stack Overflow
- Reddit programming communities
- C++ developer mailing lists

Use search engines with descriptive queries including the terms `GoogleTest` or `gMock` alongside your error or problem description for targeted results.


## Reporting Bugs and Feature Requests

- Always check the [Changelog](https://github.com/google/googletest/blob/main/CHANGELOG.md) to see if your issue has been addressed in recent versions.
- Include snippets of your test code and build configuration when reporting bugs.
- For security-related issues, refrain from posting publicly, and instead look for dedicated security contact information in the GitHub repository.


## Additional Help and Escalation

If you have explored the above resources without success:

- Ask questions on Stack Overflow or GitHub discussions (if enabled).
- Review the [FAQ sections](https://github.com/google/googletest/blob/main/docs/faq.md) for common problems.
- Consider filing a detailed GitHub issue with all relevant details.


## Tips for Effective Support Requests

- Include minimal reproducible examples to illustrate your problem.
- Clearly identify the problem context and what you have tried so far.
- Provide environment details: OS, compiler version, GoogleTest version.
- Use the appropriate tags or labels when posting questions or issues.
