# How can I contribute or find community-created documentation?

GoogleTest and GoogleMock thrive not only through official documentation but also via vibrant contributions and resources created by the community. This page guides you on how to contribute your own documentation, find community-driven tutorials and guides, and explore extensions that can enhance your use of GoogleTest and GoogleMock.

---

## Contributing Documentation

Contributing to GoogleTest's documentation is a valuable way to support the community and improve the framework. Whether you're enhancing existing pages, adding new tutorials, or creating examples, your contribution helps fellow users get more from the tools.

### Ways to Contribute

- **Create Tutorials and Guides:** Share your expertise by writing how-to guides or walkthroughs that clarify complex concepts or usage patterns.
- **Document Extensions and Customizations:** If you've developed useful custom matchers, actions, or test helpers, contribute documentation to help others leverage these techniques.
- **Report and Improve Clarity:** Suggest improvements, clarify explanations, or fix inaccuracies in existing documentation.

### How to Get Started

1. **Fork the Repository:** Start by forking the official GoogleTest GitHub repository.
2. **Edit Markdown Files:** Documentation is primarily in Markdown (`.md` or `.mdx`) files inside the `docs` directory.
3. **Follow Style and Structure:** Align with the tone, formatting, and style of existing docs. Use clear, user-centric language that aids understanding.
4. **Test Your Changes:** Preview and test the documentation locally or via built-in GitHub preview features.
5. **Submit a Pull Request:** Provide a concise description of your changes, referencing related documentation or user needs.

For detailed contributor guidelines, please refer to the repository's contributing instructions.
