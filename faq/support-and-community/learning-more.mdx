## Common Issues & Gotchas

- **Uninteresting Calls**: By default, calls to mock methods without explicit expectations generate warnings. To suppress or control these, use `NiceMock` or `StrictMock` wrappers.
- **Over-Specifying Expectations**: Setting overly strict expectations can cause brittle tests. Use `ON_CALL` for default behaviors and reserve `EXPECT_CALL` for essential interaction verification.
- **Order of EXPECT_CALL Statements**: More specific expectations should be declared after more general ones, as gMock matches calls against expectations in reverse declaration order.
- **Mock Object Destruction**: Ensure mock objects have virtual destructors to avoid resource leaks.
- **Testing Move-Only Types**: Use lambdas or proper `WillOnce`/`WillRepeatedly` with move semantics to mock methods involving move-only parameters or return types.

## Error Messages & Debugging Tips

- Use the `--gmock_verbose=info` flag to see detailed traces of mock calls and matching expectations.
- Warnings about uninteresting calls indicate calls to mock methods without set expectations.
- Errors regarding unexpected calls mean that a call was made that doesn't satisfy any active expectation.
- Use sequences and the `.After()` clause to control invocation order and diagnose failures related to call sequences.

---

## Troubleshooting Procedures

1. Verify the environment matches the System Requirements (C++17, compiler support).
2. Confirm mock objects have properly defined expectations before exercising them.
3. Use `NiceMock` to silence warnings during initial test development.
4. Increase verbosity with flags to get stack traces and detailed call matching.
5. Ensure all mocks with expectations are destructed or explicitly verified using `Mock::VerifyAndClearExpectations`.

## Performance & Optimization

- Write efficient mocks by minimizing unneeded expectations.
- Organize test suites logically to enable selective and parallel test execution.
- Use catch-all `EXPECT_CALL` with `.Times(AnyNumber())` to reduce overhead.

## Example: Accessing Further Documentation from Your Code

```cpp
#include <gmock/gmock.h>

using ::testing::NiceMock;
class MockFoo : public Foo {
 public:
  MOCK_METHOD(int, GetSize, (), (const, override));
  MOCK_METHOD(void, DoSomething, (), (override));
};

// When your test needs to suppress warnings on uninteresting calls:
NiceMock<MockFoo> mock_foo;
EXPECT_CALL(mock_foo, DoSomething());

// For action details and advanced mock usage, visit:
// https://github.com/google/googletest/blob/main/docs/gmock_cook_book.md
```
