# Where can I get support or contribute to GoogleTest?

GoogleTest is a widely used C++ testing framework, supported by a vibrant community and an active development team. Whether you're seeking help, looking to contribute, or wanting to stay connected with the latest updates, this page directs you to the most helpful resources.

---

## Getting Help and Reporting Issues

### Official Documentation

The primary source for comprehensive and up-to-date information about GoogleTest is the official documentation hosted on GitHub Pages:

- **GoogleTest Documentation:** [https://google.github.io/googletest/](https://google.github.io/googletest/)

Here, you will find the full user guide including the Primer, Advanced Topics, API Reference, and FAQ sections describing how to get started, advanced usage, and troubleshooting tips.

### Issue Tracker

If you encounter bugs, unexpected behavior, or have feature requests, you should report them in the official GitHub issue tracker:

- **GoogleTest Issues on GitHub:** [https://github.com/google/googletest/issues](https://github.com/google/googletest/issues)

Before opening a new issue, please search for existing reports to avoid duplicates and help maintain project clarity.

### Community Discussions

Community support and discussions can be found through various avenues such as Google groups, Stack Overflow, and other developer forums:

- **Stack Overflow:** Tag your questions with `googletest` for community assistance.
- **Google Groups or Mailing Lists:** Check the GoogleTest community pages for mailing list links.

Engaging community members and contributors can provide practical advice and solutions.

## Contributing to GoogleTest

GoogleTest is an open-source project and welcomes contributions ranging from bug fixes and documentation improvements to new feature implementations.

### Contribution Guidelines

To contribute effectively, follow the official guidelines:

- Read the [`CONTRIBUTING.md`](https://github.com/google/googletest/blob/main/CONTRIBUTING.md) file in the repository.
- Follow the coding style and testing requirements outlined.
- Ensure your code passes all relevant tests.
- Use pull requests and provide clear descriptions.

This ensures smooth integration and maintains the quality and reliability of GoogleTest.

### Code Repository

The source code and the development history for GoogleTest are available on GitHub:

- **GoogleTest GitHub Repository:** [https://github.com/google/googletest](https://github.com/google/googletest)

Forking this repository allows you to build, test, and experiment with the GoogleTest codebase before contributing.

## Ecosystem and Related Projects

GoogleTest integrates with a rich ecosystem of related tools and extensions. Exploring these can enhance your testing workflows:

### Related Open Source Projects

- **GoogleMock:** Google's framework for C++ mocking, integrated with GoogleTest.
  - [GoogleMock Documentation](https://github.com/google/googletest/tree/main/googlemock)

- **GTest Runner:** GUI-based test runner for Windows and Linux.

- **GoogleTest UI:** Visual test runner with progress tracking.

- **GTest TAP Listener:** Event listener supporting test output in TAP format.

- **GTest-Parallel:** Runs test binaries in parallel to speed up test suites.

- **VS Code Extensions:** GoogleTest Adapter and C++ TestMate for integrated IDE support.

- **Cornichon:** Gherkin DSL parser generating GoogleTest stubs.

### Extensions and Helpers

Many of these projects help customize output, enable parallel test execution, or add better IDE integration, making daily testing easier and more productive.
