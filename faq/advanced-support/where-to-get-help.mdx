# Where Can I Get Help and Find Community Resources?

GoogleTest is a widely-used C++ testing framework with a robust ecosystem of support options and community channels. This page acts as a central hub, guiding you to official support channels, active community forums, essential documentation, and providing clear instructions on reporting issues or contributing to the project’s ongoing improvement.

---

## Official Support Channels

### GitHub Repository

- **Location:** [https://github.com/google/googletest](https://github.com/google/googletest)
- The primary source for the latest code, releases, and issue tracking.
- Use the GitHub **Issues** feature to report bugs or suggest enhancements.
- Pull requests are welcome for contributions following the contribution guidelines.

### Documentation Website

- **Location:** [https://google.github.io/googletest/](https://google.github.io/googletest/)
- Official, up-to-date documentation covering installation, usage, API reference, guides, and advanced topics.
- This is your go-to resource for understanding the framework features and best practices.

### Mailing Lists / Google Groups

- While there is no dedicated official mailing list, many community members use Google Groups or mailing lists for GoogleTest to ask questions and share tips. [Look for community-run GoogleTest forums](https://groups.google.com/g/googletest).

### Issue Tracker Etiquette

- When filing an issue:
  - Search for existing issues to avoid duplicates.
  - Provide detailed descriptions, including steps to reproduce, environment, version information, and minimal reproducible examples.
  - Specify platform, compiler, and build settings.

<Tip>
Clear and well-formed issue reports help maintainers respond efficiently and fix problems faster.
</Tip>

## How to File Issues or Contribute

### Filing Issues

Follow these steps to get your bugs or feature requests addressed:

1. **Search First:** Check the existing open and closed issues.
2. **Use the Template:** Fill in reproduction steps, platform details, compiler versions, and logs.
3. **Provide Minimal Code:** A minimal reproducible example accelerates diagnosis.
4. **Describe Expected vs Actual:** Clarify what behavior you anticipated vs. what occurred.

### Contributing Code or Documentation

- **Fork the Repository:** Work with your own branch on [GitHub](https://github.com/google/googletest).
- **Follow Coding Conventions:** Respect style, naming, and testing practices indicated in the codebase.
- **Write Tests:** Contributions should include tests demonstrating fixes or new functionality.
- **Submit a Pull Request:** Provide a clear description, link to related issues, and ensure all tests pass.

### Community Contribution Guidelines

- GoogleTest encourages high standards of code quality and maintenance.
- Contributions improving docs, examples, or tools are highly valued.
- Review the `CONTRIBUTING.md` file in the repo for detailed guidelines.

<Tip>
Consistent collaboration with maintainers and adherence to contribution standards increases the chance of successful merges.
</Tip>
