# Where can I get help or report issues?

This page guides you to the official support channels, community forums, issue trackers, and documentation resources for GoogleTest and GoogleMock. It also provides tips on how to ask effective questions or file informative bug reports to get timely and helpful responses.

---

## Official Support Channels

### 1. GitHub Issue Tracker

The primary location for reporting bugs, feature requests, or unexpected behavior is the official GitHub repository:

- **GoogleTest & GoogleMock Repository:** [https://github.com/google/googletest/issues](https://github.com/google/googletest/issues)

Before opening a new issue:

- Search existing issues to avoid duplicates.
- Provide a clear and concise description.
- Include steps to reproduce the problem.
- Attach relevant code snippets or minimal reproducible examples.
- Specify your environment details such as OS, compiler version, and GoogleTest/GoogleMock version.

### 2. Mailing Lists and Google Groups

GoogleTest maintains a developer mailing list where users and contributors discuss usage questions and development:

- [googletestframework@googlegroups.com](https://groups.google.com/g/googletestframework)

Use the mailing list to ask usage questions and seek community support.

### 3. Stack Overflow

For broader community support and practical help, many users ask questions tagged with `googletest` and `googlemock` on Stack Overflow:

- [https://stackoverflow.com/questions/tagged/googletest](https://stackoverflow.com/questions/tagged/googletest)
- [https://stackoverflow.com/questions/tagged/googlemock](https://stackoverflow.com/questions/tagged/googlemock)

When posting, provide detailed context, error messages, and minimal examples to receive effective answers.

## Tips for Effective Support Requests

To get timely and helpful responses whether on GitHub issues, mailing lists, or Stack Overflow:

- **Be precise and clear:** Summarize what you are trying to do and what is not working.
- **Include minimal reproducible examples:** Provide a small code snippet or project that reproduces the issue.
- **Specify your environment:** OS, compiler, GoogleTest/GoogleMock versions.
- **Share error messages and logs:** Exact failure outputs help diagnose faster.
- **Check for duplicates:** Search existing issues or questions before posting.
