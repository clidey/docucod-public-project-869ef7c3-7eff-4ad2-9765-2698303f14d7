# Where can I get more help or contribute?

This page serves as your central directory for finding support resources, official documentation links, community forums, and guidelines on contributing to GoogleTest and GoogleMock. Whether you have questions, want to report bugs, seek feature enhancements, or simply engage with the community, the resources listed here will guide you to the right channels.

---

## Support Channels

- **Official GitHub Repository:**
  - For bug reports, feature requests, and source access, visit the official repository:
    [https://github.com/google/googletest](https://github.com/google/googletest)
  - Use GitHub Issues to submit bug reports or request features.
  
- **GoogleTest Documentation:**
  - The primary source of detailed information and references:
    [GoogleTest Documentation](https://google.github.io/googletest/)
  - Contains user guides, API references, best practices, and FAQs.

- **Community Forums and Q&A Sites:**
  - Stack Overflow with the `googletest` tag: [https://stackoverflow.com/questions/tagged/googletest](https://stackoverflow.com/questions/tagged/googletest)
  - Google Groups or mailing lists (if any up-to-date relevant ones exist).

- **Issue Tracking and Reporting:**
  - Report issues via GitHub Issues on the official repository.
  - Before reporting, check existing issues for duplicates to avoid fragmentation.

- **Contribution Support:**
  - Please refer to the contribution guidelines in the repository for instructions on how to submit patches or participate in development.

## Contribution Guidelines

To contribute effectively, familiarize yourself with the guidelines and workflows:

- **Code Contributions:**
  - Fork the repository and submit pull requests following the project style.
  - Write clear commit messages and accompany code changes with tests.
  
- **Issue Reporting:**
  - Provide detailed descriptions and reproduction steps.
  - Include relevant logs or error messages.

- **Documentation Contributions:**
  - You can contribute by suggesting improvements, adding examples, or enhancing guides.
  - Follow the documentation style and markup conventions.

For detailed contribution information, check the repository's `CONTRIBUTING.md` file or relevant documentation in the project root.

---

## Getting the Most Out of Support

- **Search First:** Most common issues and questions are answered in the official documentation and FAQs.
- **Reproducible Examples:** When seeking help, provide minimal, complete, and verifiable examples.
- **Stay Updated:** Always use the latest stable release or branch to avoid already resolved issues.
- **Report Bugs Respectfully:** Provide detailed, clear, and polite reports to help maintainers respond effectively.
