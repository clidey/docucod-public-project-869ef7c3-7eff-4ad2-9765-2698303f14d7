# Where can I get help or contribute to GoogleTest?

This page provides essential information on how to seek help, report issues, engage with the GoogleTest community, and contribute to the GoogleTest codebase. Whether you are encountering problems, looking for peer support, or interested in improving the project by submitting patches, the resources below will guide you in connecting with the GoogleTest ecosystem.

---

## Reporting Issues

If you experience bugs, unexpected behaviors, or have feature requests, the best way to report them is through the official GitHub issue tracker:

- **GitHub Issues:** [https://github.com/google/googletest/issues](https://github.com/google/googletest/issues)

When filing an issue, please include:

- Clear and concise description of the problem or feature.
- Steps to reproduce, including relevant code snippets or test cases.
- Environment details such as platform, compiler version, and GoogleTest version.
- Any relevant error messages or outputs.

Providing thorough details helps maintainers respond quickly and effectively.


## Getting Peer Assistance

If you need help or advice beyond filing issues, the community provides multiple venues for discussion and learning:

- **Stack Overflow:** Tag your question with [`googletest`](https://stackoverflow.com/questions/tagged/googletest) to ask about usage and troubleshooting.
- **Google Groups:** The `googletest` mailing list offers a place for community interaction and informal discussions.
- **Community-Created Documentation:** Various user-generated guides and tips can be found, like [Googlemock Insights by ElectricRCAircraftGuy](https://github.com/ElectricRCAircraftGuy/eRCaGuy_dotfiles/blob/master/googletest/insights.md).

These resources complement the official documentation and are suitable for getting answers, sharing knowledge, and connecting with other users.

## Contributing to the Codebase

GoogleTest welcomes contributions from the community. Whether you're fixing bugs, adding features, improving documentation, or optimizing performance, here's how you can contribute responsibly and efficiently:

### Contributing Guidelines

- Review the **[CONTRIBUTING.md](https://github.com/google/googletest/blob/main/CONTRIBUTING.md)** file in the repository for detailed instructions.
- Follow the project's coding style and testing requirements.
- Submit pull requests against the `main` branch.
- Provide clear commit messages and descriptions.
- Include tests or documentation updates that accompany your code changes.

### Best Practices

- Stay in sync with the project by frequently pulling the latest changes.
- Use the existing test suites to validate your modifications.
- Make use of CMake or Bazel build configurations as appropriate for testing your changes.
- Engage respectfully with maintainers and other contributors during code reviews.

### Developer Resources

- Source code is hosted on GitHub: [https://github.com/google/googletest](https://github.com/google/googletest)
- The repository includes a variety of build scripts, tests, and examples in C++.
- See the `README.md` and other internal documentation for build and test instructions.
