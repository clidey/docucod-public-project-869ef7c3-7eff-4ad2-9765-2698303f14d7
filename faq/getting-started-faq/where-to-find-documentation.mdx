# Where Can I Find Documentation & Examples?

Discovering the right documentation and practical examples swiftly is key to mastering GoogleTest and GoogleMock. This page guides you through the official documentation landscape and points you to comprehensive guides, detailed references, and ready-to-use sample code that address common needs and sharpen your testing skills.

---

## Navigating the Official Documentation

GoogleTest and GoogleMock are extensively documented with organized, user-focused resources. Start by understanding how the documentation is structured and where to jump based on what you want to achieve:

- **User Guides and Tutorials:** Step-by-step instructions that walk you through setting up, writing tests, creating mocks, and more.
- **API References:** Detailed documentation of macros, classes, functions, and usage patterns.
- **Concepts and Architecture:** Deep dives into core principles, system architecture, and extensibility.
- **FAQs:** Common questions and troubleshooting tips from real users.

Each documentation page usually includes practical examples and usage notes to help you get a hands-on understanding.

## Where to Find Key Guides and References

Here are the primary categories of documentation and recommended entry points:

### 1. Getting Started Guides

- **Setup and Quickstart:** Learn how to obtain and install GoogleTest and GoogleMock, configure your project, and write your first tests. (_See: [Setting Up and Running Your First Test](https://google.github.io/googletest/guides/getting-started/setup-quickstart.html)_) 
- **Writing Tests:** Understand the syntax for writing test cases, fixtures, and test discovery. (_See: [Writing and Discovering Tests](https://google.github.io/googletest/guides/getting-started/writing-and-discovering-tests.html)_) 
- **Mocking Basics:** Learn how to define mock classes and set expectations on mock methods. (_See: [Mocking for Dummies](https://google.github.io/googletest/gmock_for_dummies.html) and [Creating and Using Basic Mocks](https://google.github.io/googletest/guides/mocking-scenarios/creating-basic-mocks.html)_)

### 2. Core Features and Assertions

- Comprehensive explanation of assertion macros, fatal vs non-fatal assertions, and how to write custom assertions. (_See: [Assertions Reference](https://google.github.io/googletest/reference/assertions.html) and [Making Assertions: Core and Custom](https://google.github.io/googletest/guides/getting-started/assertions-basics.html)_) 

### 3. Advanced Mocking Patterns

- Detailed recipes for complex mock scenarios such as delegate to fakes, strictness management, and expectations ordering. (_See: [gMock Cookbook](https://google.github.io/googletest/gmock_cook_book.html) and [Mock Strictness: Nice, Naggy, Strict](https://google.github.io/googletest/guides/mocking-scenarios/mock-strictness-best-practices.html)_) 

### 4. API References

- Full specifications of all relevant macros, matchers, actions, and mock-related utilities. (_See: [Mocking Reference](https://google.github.io/googletest/reference/mocking.html), [Matchers Reference](https://google.github.io/googletest/reference/matchers.html), [Actions Reference](https://google.github.io/googletest/reference/actions.html)_) 

### 5. FAQ and Troubleshooting

- Answers to frequent questions, tips on setup issues, and strategies for debugging mocks and tests. (_See: [Legacy gMock FAQ](https://google.github.io/googletest/gmock_faq.html) and [Getting Started FAQ](https://google.github.io/googletest/faq/getting-started-faq/where-to-find-documentation.html)_) 

## Examples and Sample Code

Practical learning is accelerated by examining well-crafted test and mock examples:

- **GoogleTest and GoogleMock Examples:** Found in the samples directory of the repository, these include representative usage scenarios.
- **Code Snippets in Guides:** Nearly all guides and references embed small, focused code snippets demonstrating key usage.
- **Cheat Sheets:** Quick reference sheets highlight common patterns and syntax, perfect for day-to-day use. (_See: [gMock Cheat Sheet](https://google.github.io/googletest/gmock_cheat_sheet.html)_) 

These examples cover basic mock classes, setting expectations, handling overloads, and using matchers and actions effectively.

## How to Access the Documentation and Examples

1. **Official Website:** Visit the GoogleTest official website ([https://google.github.io/googletest/](https://google.github.io/googletest/)) where all guides, API refs, and FAQs are hosted with up-to-date content.
2. **Repository Samples:** Explore the [googletest GitHub repository](https://github.com/google/googletest) which contains sample test code you can clone and adapt.
3. **Offline Access:** Consider downloading the docs or browsing via the repository's `docs` directory for offline study.

## Tips for Efficiently Finding What You Need

- Use the **Navigation Map** on the website to jump quickly between guides, API references, and FAQs relevant to your current task.
- When searching for how to mock specific function types or complex scenarios, refer to the **gMock Cookbook** for tested recipes and examples.
- For quick syntax reminders or reference, the **Cheat Sheets** are invaluable.
- Troubleshooting setup or runtime issues is expedited with the **Common Setup Issues** and **Legacy FAQ** pages.

### Example: Finding How to Define a Mock Class

Suppose you want to define a mock class for an interface `Foo`. Start by reviewing the [gMock for Dummies](https://google.github.io/googletest/gmock_for_dummies.html) guide and the [gMock Cheat Sheet](https://google.github.io/googletest/gmock_cheat_sheet.html) for syntax examples such as:

```cpp
class MockFoo : public Foo {
 public:
  MOCK_METHOD(int, GetSize, (), (const, override));
  MOCK_METHOD(std::string, Describe, (const char* name), (override));
};
```

Then, explore `EXPECT_CALL` syntax and `ON_CALL` behaviors in the [Mocking Reference](https://google.github.io/googletest/reference/mocking.html).
