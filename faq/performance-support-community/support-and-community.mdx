# Where can I get further help or contribute?

This page provides essential resources for users who need additional support with GoogleTest and GoogleMock or who wish to contribute to the project. Whether you are troubleshooting, looking to engage with the community, or ready to contribute code or documentation, the following guidance and links will direct you to the right places.

---

## Frequently Asked Questions

### How can I get help with issues I encounter?

- **GitHub Issues**: The primary place to report bugs, request features, or ask for help is the [GoogleTest GitHub Issues](https://github.com/google/googletest/issues). Make sure to search existing issues before opening a new one to avoid duplicates.

- **Community Forums**: While GoogleTest does not have an official forum, platforms like Stack Overflow provide community support. Use the tags [googletest](https://stackoverflow.com/questions/tagged/googletest) and [googlemock](https://stackoverflow.com/questions/tagged/googlemock) for relevant discussions.

- **Email Lists/Chat**: Currently, there are no official mailing lists or chat channels. Community interaction is mostly through GitHub and third-party sites.

### Where can I find extended or community-created documentation?

- [gMock for Dummies](https://google.github.io/googletest/gmock_for_dummies.html): Friendly tutorial and extensive examples for writing mocks.
- [Legacy gMock FAQ](https://google.github.io/googletest/gmock_faq.html): Answers to common questions and practical advice.
- [gMock Cookbook](https://google.github.io/googletest/gmock_cook_book.html): Recipes and advanced usage patterns.

These community resources often contain insights beyond the official reference and are invaluable for tackling real-world scenarios.

### How do I contribute to GoogleTest or GoogleMock?

GoogleTest welcomes contributions from the community including bug fixes, new features, documentation improvements, and test cases.

- **Contribution Guidelines**: Please read the official [Contributing Guide](https://github.com/google/googletest/blob/main/CONTRIBUTING.md) to understand the projectâ€™s workflow, coding standards, and review process.

- **Submitting Changes**:
  1. Fork the repository.
  2. Create a feature branch.
  3. Write clear commit messages.
  4. Include tests for new features or bug fixes.
  5. Submit a pull request against the `main` branch.

- **Code of Conduct**: Adhere to the [Google Open Source Code of Conduct](https://opensource.google/codeofconduct/), which fosters a welcoming and respectful environment.

- **Helping with Documentation**: Improvements to documentation writ large are highly valued. Consider updating guides, fixing typos, or enhancing examples. Documentation lives in this repository and is often maintained in Markdown format.

### Are there official tools or plugins integrating with IDEs?

- **Visual Studio Code Extensions**:
  - [GoogleTest Adapter](https://marketplace.visualstudio.com/items?itemName=matepek.vscode-catch2-test-adapter): Supports running and debugging GoogleTest tests from VS Code.
  - [CTest Explorer](https://marketplace.visualstudio.com/items?itemName=matepek.vscode-cmake-test-adapter): Integrates CMake-based test frameworks including GoogleTest.

- **Other Tools**:
  - Various third-party GUI test runners, CI plugins, and continuous integration scripts are available. See the [Third-Party Integrations & Ecosystem](https://github.com/google/googletest#ecosystem) section on the official GitHub for curated lists.

### How can I stay updated on releases and community developments?

- **Release Notes**: Monitor the [Release Notes](https://github.com/google/googletest/releases) page on GitHub for the latest features, fixes, and announcements.

- **Changelog**: The changelog in the repository provides a history of changes and upgrade guidance.

- **Community Contributions**: Keep an eye on pull requests and issue discussions to learn about ongoing work.

## Troubleshooting & Support

If you run into common problems, these steps can expedite resolution:

1. **Search existing issues and documentation** to check if your problem is already addressed.
2. **Enable verbose output** with the `--gmock_verbose=info` flag to get detailed traces.
3. **Provide minimal reproducible examples** when asking for help.
4. **Check platform compatibility** with the [Supported Platforms & Integrations](../overview/introduction_and_value/supported_platforms_and_integration.md) guide.
5. **Use debugging tools** outlined in the [Error Handling & Status Codes](../api-reference/gtest-core-apis/error-handling.md) documentation.
