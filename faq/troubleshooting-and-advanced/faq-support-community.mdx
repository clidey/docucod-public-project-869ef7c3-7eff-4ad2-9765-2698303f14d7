# Getting Help: Support Resources & Community Channels

When working with GoogleTest and GoogleMock, accessing the right support channels and community resources can be essential to resolving issues, learning best practices, and contributing to the project. This document guides you through official support options, community forums, documentation access, issue reporting, and contribution guidelines.

---

## Support Channels

### Official GoogleTest Support

- **Issue Tracker:** Report bugs, request features, or check the status of known issues on the [GoogleTest GitHub Issues page](https://github.com/google/googletest/issues).
- **Stack Overflow:** Ask technical questions tagged with [`googletest`](https://stackoverflow.com/questions/tagged/googletest) to get help from the wider developer community.
- **Mailing Lists & Forums:** While GoogleTest does not maintain an official mailing list, various community-run groups are available and searchable.

### Getting Professional Support

- Organizations using GoogleTest extensively may seek professional support or consultancy services from third parties specializing in C++ testing.

## How to Report Issues Effectively

Providing clear, actionable reports accelerates problem resolution:

- **Reproduce Steps:** Describe precise steps to replicate the issue.
- **Environment Details:** Include system, compiler, and GoogleTest version info.
- **Minimal Example:** Share concise code snippets or test cases demonstrating the problem.
- **Expected vs Actual Behavior:** Clarify what you expected and what actually happened.
- **Logs & Error Messages:** Attach outputs, warnings, or error messages.

Refer to [Contributing Guidelines](https://github.com/google/googletest/blob/main/CONTRIBUTING.md#bug-reports) for detailed instructions.

## How to Ask Questions or Get Community Help

- Before posting, search the existing issues and Stack Overflow questions for answers.
- When asking, provide detailed context and code samples.
- Use related tags such as `googletest` and `googlemock` to reach the right audience.

## Contributing to GoogleTest

The project welcomes contributions, including bug fixes, improvements, documentation, and examples:

- **Fork and Clone:** Start by forking the repository and cloning it locally.
- **Read Contribution Guide:** Follow the [CONTRIBUTING.md](https://github.com/google/googletest/blob/main/CONTRIBUTING.md) file.
- **Coding Standards:** Write clean, well-documented, and tested code.
- **Submit Pull Requests:** Ensure PRs are focused and include tests.
- **Participate in Code Reviews:** Engage constructively in the review process.

## Additional Tips

- Keep GoogleTest updated to leverage fixes and new features.
