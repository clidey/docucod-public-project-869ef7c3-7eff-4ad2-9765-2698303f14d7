# Accessing Support & Community

Explore the most effective channels to get assistance with GoogleTest and GoogleMock, report issues, contribute improvements, and engage with the community. This page guides you to official documentation, issue trackers, contribution guides, and forums where you can collaborate and find ongoing support.

---

## Where to Get Help

If you encounter difficulties or want to deepen your understanding of GoogleTest and GoogleMock, multiple official and community resources are available:

- **Official Documentation:** The complete set of guides and references on using GoogleTest and GoogleMock, including primers, API references, and cookbooks.
- **Issue Tracker:** Report bugs or feature requests in the [GoogleTest GitHub issues](https://github.com/google/googletest/issues).
- **Community Forums:** Engage with other users, ask questions, and share insights on platforms like Stack Overflow and Google Groups.
- **Mailing Lists and Chat:** Join the GoogleTest mailing list or chat channels recommended by the project for real-time support.

<Info>
For focused help on integration, troubleshooting, and advanced mocking techniques, consult the corresponding FAQ pages and guides linked below.
</Info>

## Reporting Bugs and Issues

When you discover unexpected behavior, crashes, or feature gaps:

1. **Search Existing Issues:** Check the [issue tracker](https://github.com/google/googletest/issues) to see if your problem has been reported.
2. **Provide Clear Details:** Include a minimal reproducible example, environment details, and the GoogleTest/GoogleMock version.
3. **Follow the Template:** Use the issue template guidelines when opening new issues to help maintainers prioritize and diagnose the problem efficiently.

<Warning>
Avoid posting questions or usage support requests in the bug tracker; use community channels for those.
</Warning>

## Contributing Documentation and Code

GoogleTest and GoogleMock welcome contributions that improve the framework, documentation, examples, and build infrastructure.

### How to Contribute

- **Fork the Repository:** Start with your own copy at [https://github.com/google/googletest](https://github.com/google/googletest).
- **Make Your Changes:** Follow coding and style guidelines documented in the repo.
- **Write Tests:** Ensure your changes include adequate test coverage when applicable.
- **Submit a Pull Request:** Provide a clear description of your changes and link any related issues.

### Areas for Contribution

- **Documentation:** Clarifications, new examples, translations, or tutorials.
- **Code Improvements:** Bug fixes, feature enhancements, or performance optimizations.
- **Sample Tests and Mock Classes:** More real-world examples to help users.
- **Platform Support and Integration:** Enhancements for CI/CD pipelines, IDEs, and build systems.

<Tip>
Consult the `CONTRIBUTING.md` in the repository and the official contribution guidelines before submitting your work.
</Tip>

## Community and Collaboration

Connecting with other users and developers can accelerate your mastery of GoogleTest and GoogleMock:

- **Stack Overflow:** Search tags like [`googletest`](https://stackoverflow.com/questions/tagged/googletest) and [`googlemock`](https://stackoverflow.com/questions/tagged/googlemock).
- **Google Testing Blog:** Stay informed on best practices and new features.
- **GoogleTest GitHub Discussions:** Join conversations about development, usage, and plans.
- **Meetups and Conferences:** Occasionally, Google testing tools are discussed in C++ and testing conferences.
