# Where can I get help and find community resources?

This page provides a comprehensive directory of official support channels, community forums, relevant documentation, and ways to contribute or report issues related to GoogleTest and GoogleMock. It is designed to help users connect with maintainers and other practitioners effectively.

---

## Frequently Asked Questions

### Where can I get official support for GoogleTest and GoogleMock?
Official support is primarily offered via the following channels:

- **GitHub Repository Issues:** The main place to report bugs, request features, and track issues is the [GoogleTest GitHub repository](https://github.com/google/googletest/issues). Maintainers actively monitor this to help the community.

- **GoogleTest and GoogleMock Documentation:** Detailed guides, API references, FAQs, and cookbooks are available on the official GitHub Pages site, providing comprehensive help:
  - [GoogleTest Documentation](https://google.github.io/googletest/)
  - [gMock for Dummies](https://google.github.io/googletest/gmock_for_dummies.html)
  - [gMock Cookbook](https://google.github.io/googletest/gmock_cook_book.html)
  - [gMock FAQ](https://google.github.io/googletest/gmock_faq.html)

- **Community Forums and Q&A Sites:** While there is no official forum, many users ask questions on platforms such as Stack Overflow using the `googletest` and `googlemock` tags. Responses from the community and sometimes contributors help solve common problems.


### Are there community-created guides or additional resources?
Yes, beyond the official documentation, community contributed content includes:

- Blog posts, tutorials, and videos explaining core concepts, mock usage patterns, and testing techniques.
- Integration examples demonstrating how to use GoogleTest with different build systems like Bazel and CMake.
- Open-source projects that use GoogleTest as a reference for advanced testing scenarios.

You can often discover such materials linked from the documentation, GitHub discussions, or by a search on developer communities.


### How do I contribute to GoogleTest and GoogleMock?
Contributions to GoogleTest and GoogleMock are welcome and encouraged. Below are practical ways to contribute:

- **Reporting Issues:** Use the GitHub Issues page to report bugs or suggest features. Make sure to provide detailed information to help maintainers understand and reproduce the problem.

- **Submitting Pull Requests:** If you want to fix bugs, add new features, or improve documentation, fork the repository, create branches for your changes, and submit pull requests following the contribution guidelines.

- **Participating in Discussions:** Engage with other users and contributors in GitHub discussions or external forums.

- **Documentation and Tutorials:** Share your knowledge by contributing to documentation improvements or creating helpful tutorials.

Refer to the [CONTRIBUTING.md](https://github.com/google/googletest/blob/main/CONTRIBUTING.md) file on GitHub for detailed instructions.


### How can I report security issues?
Security issues should be reported privately to the Google Security Team. Avoid posting them publicly on forums or GitHub issues to prevent exposure. Instructions for responsible disclosure can usually be found in the GoogleTest repository's security or contributing guidelines.


### Where can I find the source code and official documentation site?
- **Source Code:** [https://github.com/google/googletest](https://github.com/google/googletest)
- **Official Documentation & User Guide:** [https://google.github.io/googletest/](https://google.github.io/googletest/)


### Are there recommended ways to connect with other users?
Though there is no dedicated official community forum, you can:

- Look for discussions on Stack Overflow under `googletest` and `googlemock` tags.
- Join developer groups or mailing lists that focus on C++ testing.
- Follow updates and conversations on the GitHub repository.

## Best Practices for Seeking Help

- **Start with Documentation:** Before seeking help, review the official documentation sections related to your issue, such as the FAQ, troubleshooting guides, and API references.

- **Search Before Posting:** On Q&A sites or GitHub issues, search for your question first to avoid duplicates and find immediate answers.

- **Provide Reproducible Examples:** When reporting bugs or asking questions, include minimal compilable code examples, environment details, and error messages.

- **Be Specific:** Clearly articulate your problem, intended behavior, and what you have tried so far.

## How to Report Issues and File Bugs

When you encounter problems or bugs with GoogleTest or GoogleMock:

1. **Check existing issues:** Search the GitHub issues page to see if your problem has been reported.
2. **Create a minimal reproducer:** Prepare a small test case that clearly demonstrates the problem.
3. **Open a GitHub issue:** Include steps to reproduce, environment details (OS, compiler version), and expected vs actual behavior.
4. **Use appropriate labels:** Flag your issue as a bug, feature request, or question.

Maintainers often triage issues and respond to provide fixes or guidance.
