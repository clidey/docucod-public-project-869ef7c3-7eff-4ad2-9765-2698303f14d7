# Where can I get more help or report issues?

This page consolidates the primary official and community support channels for GoogleTest and GoogleMock users. Whether you're looking to report bugs, request features, or ask questions about usage, this guide directs you to the right resources efficiently. Additionally, it provides access to community-created documentation and contribution guidelines to help you engage with the project effectively.

---

## Official Support Channels

### 1. Reporting Bugs and Feature Requests

To report bugs or request features, use the GoogleTest GitHub repository's Issues page:

- **GitHub Issues:** [https://github.com/google/googletest/issues](https://github.com/google/googletest/issues)

When creating an issue:

- Provide a clear, concise title and detailed description.
- Include steps to reproduce the bug or clearly explain the feature need.
- Supply relevant code samples, error messages, and environment details (OS, compiler version).

This helps the maintainers quickly understand and address the problem.

### 2. Contribution Guidelines

If you plan to contribute code, documentation, or other improvements, please follow the project's official guidelines:

- **Contribution Guide:** [https://github.com/google/googletest/blob/main/CONTRIBUTING.md](https://github.com/google/googletest/blob/main/CONTRIBUTING.md)

This document outlines:

- How to set up your development environment.
- Code style and testing requirements.
- Pull request submission process.

Following these ensures smooth collaboration with the GoogleTest maintainers.

### 3. Asking Usage Questions

For questions about how to use GoogleTest or GoogleMock:

- Check the **GoogleTest FAQ** and **gMock FAQ:**
  - [GoogleTest FAQ](https://google.github.io/googletest/faq.html)
  - [gMock FAQ](https://google.github.io/googletest/gmock_faq.html)
- Browse or post questions on:
  - **Stack Overflow:** Tag questions with [`googletest`](https://stackoverflow.com/questions/tagged/googletest) or [`google-mock`](https://stackoverflow.com/questions/tagged/google-mock)
  - **GitHub Discussions:** If enabled, this is a good place for community discussion.

Before posting questions, it's best to search these resources to avoid duplicates.

## Tips for Getting Effective Help

- When reporting issues, always provide minimal reproducible examples.
- Search existing issues and Q&A before posting a new query.
- Use clear and precise language.
- Follow project etiquette and contribution guidelines to foster a positive community experience.
