# Where to Get Help and Report Issues

This page guides you to the official channels for finding the latest documentation, reporting bugs, engaging with the community, and contributing to GoogleTest and GoogleMock. Whether you are looking for up-to-date information, need to report an issue, or want to request new features, this page points you to the right resources.

---

## Official Documentation

To fully understand and leverage GoogleTest and GoogleMock, rely on the official documentation hosted and maintained by the project team. These include comprehensive guides, references, and cookbooks written to cover every aspect of usage, from basic testing to complex mocking scenarios.

- **Primary Documentation:**
  - [gMock for Dummies](https://google.github.io/googletest/gmock_for_dummies.html) — Beginner-friendly introduction and usage patterns.
  - [Legacy gMock FAQ](https://google.github.io/googletest/gmock_faq.html) — Answers to frequent questions and common troubleshooting tips.
  - [gMock Cookbook](https://google.github.io/googletest/gmock_cook_book.html) — Detailed usage recipes for common and advanced tasks.
  - [gMock Cheat Sheet](https://google.github.io/googletest/gmock_cheat_sheet.html) — Quick reference for syntax and common patterns.

These documents provide step-by-step instructions, examples, and best practices to help you succeed with your tests.

## Bug Reporting and Feature Requests

If you encounter bugs, unexpected behavior, or have ideas for improving GoogleTest or GoogleMock, the project welcomes your input.

- **Issue Tracker:** Submit bugs and feature requests through the official GitHub repository:
  - [GitHub Issues](https://github.com/google/googletest/issues)

### Best Practices for Reporting Issues

1. **Check Existing Issues:** Before creating a new issue, search the tracker to see if your problem or idea is already noted.
2. **Provide Details:** Include clear, reproducible steps, environment information, error messages, and minimal examples if possible.
3. **Be Specific:** Clarify whether the problem is with GoogleTest, GoogleMock, or the integration.
4. **Follow Templates:** Use issue templates where provided to organize information effectively.

<Tip>
Clear, detailed bug reports help maintainers respond faster and increase the chances of a quick fix.
</Tip>

## Contributing to GoogleTest and GoogleMock

The project encourages contributions, from documentation improvements to code patches.

- **Contribution Guidelines:** Please refer to the repository README and CONTRIBUTING.md files in [the GitHub repository](https://github.com/google/googletest).
- **How to Contribute:**
  1. Fork the repository.
  2. Create feature or bugfix branches.
  3. Follow coding conventions as outlined in the documentation.
  4. Submit pull requests with clear descriptions and test coverage.

Contributions help the project evolve and meet user needs more effectively.

- Use the verbose command-line flags (`--gmock_verbose=info`) to collect detailed debug information when filing bug reports.
