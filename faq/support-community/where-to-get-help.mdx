# Where to Get Help

GoogleTest offers a rich array of support resources to guide and assist you when you encounter questions or issues. Whether you're just starting out or need help with an advanced problem, these channels connect you with comprehensive documentation, active discussions, and the vibrant community.

---

## Official Documentation

Start with GoogleTest’s official documentation—it’s extensive, well-structured, and updated with the latest guidance:

- **User's Guide**: Step-by-step tutorials from introductory to advanced topics.
- **API References**: Detailed descriptions of classes, macros, and functions.
- **FAQ Sections**: Solutions for common issues, usage tips, and best practices.

Access the docs here:
- Online: [GoogleTest Documentation](https://github.com/google/googletest/blob/main/docs/index.md)
- In the repository under `/docs`

Use these documents to find authoritative answers for setup, test writing, mocking, assertions, parameterized tests, death tests, and more.

---

## Discussion Forums and Q&A

Engage with other developers and GoogleTest contributors:

- **GitHub Discussions**: Explore discussions or ask questions about usage, features, or troubleshooting.
- **Stack Overflow**: Use the `googletest` tag to search previous answers or ask new questions.

These platforms help you tap into community wisdom, learn best practices, and troubleshoot unusual scenarios.

---

## GitHub Issues

For bugs, feature requests, or enhancement suggestions:

- Visit the [GoogleTest GitHub Issues](https://github.com/google/googletest/issues) page.
- Check if your issue has been reported or resolved.
- If not, create a detailed issue including your environment, usage scenario, and steps to reproduce.

Keep in mind: Issues are intended for product-level problems, not general usage questions.

---

## Community-Created Guides and Tutorials

Many contributors and users have created helpful tutorials, blog posts, and sample code:

- Explore examples showcasing GoogleTest usage patterns and best practices.
- Look for guides on organizing test suites, advanced mocking, or performance tuning.

These can supplement official docs with practical insights and user experience.

---

## Getting the Most from Support

### Tips:

- Always check the official documentation first to save time.
- Search existing discussions and issues before opening a new one.
- When asking for help, include context such as platform, compiler, GoogleTest version, and relevant code snippets.
- Engage respectfully and patiently with community members.

### When you need formal support or want to contribute:

- Follow the contribution guidelines in the GitHub repository.
- Subscribe to GoogleTest's release notes and changelog to stay updated.
