# Where to Get Help

This page guides you to the essential resources for getting help with GoogleTest and GoogleMock. Whether you face advanced technical issues, need to report bugs, seek community support, or plan to contribute to the project, here you'll find authoritative links and tips to navigate the support ecosystem efficiently.

---

## 1. Official Support Channels

### GoogleTest Issue Tracker

The primary venue for reporting bugs, requesting new features, or tracking ongoing issues is the GitHub issue tracker:

- GitHub Issues: [https://github.com/google/googletest/issues](https://github.com/google/googletest/issues)

Before opening a new issue, please:

- Search existing issues to check if your problem has already been reported or addressed.
- Provide a detailed description including minimal reproducible code, platform details, and exact error messages.


### Mailing Lists and Google Groups

While GoogleTest does not maintain an official mailing list, questions tagged with `googletest` or `gmock` on various community forums can be valuable:

- Stack Overflow: Use tags [googletest](https://stackoverflow.com/questions/tagged/googletest) and [gmock](https://stackoverflow.com/questions/tagged/gmock)
- Reddit communities related to C++ or testing frameworks

These community-driven channels provide peer support and diverse usage scenarios.


### GoogleTest GitHub Repository

You can also seek help by examining issues, pull requests, and discussions in the source code repository:

- Repository: [https://github.com/google/googletest](https://github.com/google/googletest)

Exploring the source and ongoing development gives insight into framework internals and recent fixes.


## 2. Documentation & Learning Resources

### Official GoogleTest Documentation

Comprehensive official resources include:

- [User's Guide](https://google.github.io/googletest/primer.html): Start here for learning basic and advanced testing techniques.
- [gMock for Dummies](https://google.github.io/googletest/gmock_for_dummies.html): Practical introduction to mocking in GoogleTest.
- [GoogleTest FAQ](https://google.github.io/googletest/faq.html): Answers to commonly encountered questions.
- [gMock Cookbook](https://google.github.io/googletest/gmock_cook_book.html): In-depth recipes for advanced mocking patterns.


### Community-Created Documentation

Community members have shared insightful guides and notes. For example:

- [Googlemock Insights](https://github.com/ElectricRCAircraftGuy/eRCaGuy_dotfiles/blob/master/googletest/insights.md)

These can provide real-world examples and deeper dives into usage.


### Tutorials and Samples

GoogleTest provides sample test code illustrating best practices and usage scenarios:

- [GoogleTest Samples](https://github.com/google/googletest/tree/main/googletest/samples)

Studying these examples helps quickly understand how to write tests and mocks effectively.


## 3. Bug Reporting and Feature Requests

To report a bug or suggest a feature:

- Use the [GitHub Issue Tracker](https://github.com/google/googletest/issues).
- Provide clear and complete reproduction steps including environment details, minimal code snippets, exact compiler versions, and logs.

Keeping descriptions concise but thorough accelerates resolution and helps maintainers prioritize work effectively.


## 4. Contributing to GoogleTest

GoogleTest encourages contributions and offers guidance for contributors:

- See [CONTRIBUTING.md](https://github.com/google/googletest/blob/main/CONTRIBUTING.md) in the GitHub repository.
- Follow coding conventions, write tests for your contributions, and ensure backward compatibility.
- Engage by reviewing open pull requests and issues.

Contributions improve the framework and strengthen community support.


## 5. Additional Tips

- When seeking help, always specify your platform, compiler, exact GoogleTest version, and minimal example code.
- Use GoogleTest's debugging and verbose logging options to gather diagnostic information:
  - `--gtest_verbose=info`
  - `--gmock_verbose=info`
- Refer to the [FAQ on Common Issues](https://google.github.io/googletest/faq.html) for immediate guidance on frequent problems.
