# Where to Find Help

This page guides you to the official resources, community forums, and support channels where you can get assistance with GoogleTest and GoogleMock. Whether you're encountering issues, seeking guidance, or wanting to contribute, these resources will connect you with the right information and people.

---

## Official Documentation

- **GoogleTest User's Guide:** Comprehensive documentation covering all GoogleTest features, including writing tests, assertions, and test lifecycle.
  - Access it at: [GoogleTest User's Guide](https://google.github.io/googletest/primer.html)

- **gMock for Dummies:** A beginner-friendly introduction to GoogleMock, with practical examples and explanations.
  - Find it here: [gMock for Dummies](https://google.github.io/googletest/gmock_for_dummies.html)

- **gMock Cookbook:** Detailed recipes on advanced mocking scenarios, matchers, actions, and best practices.
  - Available at: [gMock Cookbook](https://google.github.io/googletest/gmock_cook_book.html)

- **Mocking Reference:** The definitive reference for mock-related macros, classes, and usage.
  - See: [Mocking Reference](https://google.github.io/googletest/reference/mocking.html)

- **GoogleTest FAQ:** Answers to common questions about setup, usage, and troubleshooting.
  - Learn more: [GoogleTest FAQ](https://google.github.io/googletest/faq.html)

- **Legacy gMock FAQ:** Specific to common problems and questions about GoogleMock usage.
  - Read at: [Legacy gMock FAQ](https://google.github.io/googletest/gmock_faq.html)


## Getting Support

### GitHub Issues

- **Reporting Bugs or Feature Requests:**

  Submit issues at the official GoogleTest GitHub repository:

  [https://github.com/google/googletest/issues](https://github.com/google/googletest/issues)

  When opening an issue, please:
  - Provide a clear, minimal reproducible example.
  - Include details about your environment (OS, compiler, GoogleTest version).
  - Describe what you expected versus what happened.


### Community Forums and Q&A

- **Stack Overflow:**
  Ask questions tagged with [`googletest`](https://stackoverflow.com/questions/tagged/googletest) or [`gmock`](https://stackoverflow.com/questions/tagged/gmock).
  This is excellent for general help, best practices, and debugging advice.

- **Google Groups:**
  Engage with the Google Testing Forum for discussions and announcements.
  Subscribe and post at: [google-testing-discuss@googlegroups.com](https://groups.google.com/g/google-testing-discuss)


### Additional Resources

- **Community-Created Documentation:**
  Find insightful guides and tutorials created by users, for example the [Googlemock Insights](https://github.com/ElectricRCAircraftGuy/eRCaGuy_dotfiles/blob/master/googletest/insights.md).

- **Official Repository:**
  Always refer to the source for the latest updates and contributions at:
  [https://github.com/google/googletest](https://github.com/google/googletest)


## Tips for Effective Support Requests

- Clearly describe what you are trying to achieve.
- Include full error messages or logs.
- Share small, reproducible code snippets if possible.
- Mention versions and build environments.
- Search existing issues or questions before posting new ones.


## Escalating or Contributing

- For critical issues or security concerns, follow the procedure detailed in the GitHub repository's SECURITY.md.
- Contributing fixes or improvements is encouraged via pull requests. Read the contribution guidelines in the repository to get started.

---

Remember, effective use of these help channels and resources will accelerate your learning and problem-solving with GoogleTest and GoogleMock. Don't hesitate to explore the documentation and community resources—they are rich with practical knowledge and user experiences.
