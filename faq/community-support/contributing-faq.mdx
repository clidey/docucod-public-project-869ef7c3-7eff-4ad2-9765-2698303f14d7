# Contributing and Feedback

Welcome to the GoogleTest and GoogleMock open source community! This page answers your common questions about contributing code, improving documentation, reporting issues, and understanding the project's communication norms. It is your gateway to actively participating in making GoogleTest/GoogleMock better for everyone.

---

## Frequently Asked Questions

### How can I contribute code to GoogleTest/GoogleMock?
Contributing code is welcome! Start by reviewing [the contributing guide](https://github.com/google/googletest/blob/main/CONTRIBUTING.md) in the repository. It explains how to:

- Fork the repository and clone it locally.
- Setup your development environment.
- Follow the coding and style guidelines.
- Write tests and run existing tests before submitting changes.
- Submit your changes via a pull request for review.

Following these steps ensures smooth integration and fast feedback from maintainers.

### What types of contributions are encouraged?

- **Bug fixes:** Identify issues and submit patches that resolve them.
- **Feature additions:** Propose and implement new features aligned with project goals.
- **Documentation improvements:** Clarify explanations, update outdated info, fix typos.
- **Test enhancements:** Add more coverage or improve existing tests.

Ensure your contributions align with GoogleTest's mission of reliable, understandable, and maintainable C++ testing tools.

### How do I report an issue or bug?

If you encounter a bug, unexpected behavior, or have a feature request, please use the [Issues section](https://github.com/google/googletest/issues) on GitHub:

- Search existing issues to avoid duplicates.
- Clearly describe the problem or suggestion with steps to reproduce.
- Include environment details, versions, and relevant code snippets.

This helps the team understand and prioritize your report.

### What are the norms for communication and collaboration?

- Be respectful and polite in all interactions.
- Follow the [code of conduct](https://github.com/google/googletest/blob/main/CODE_OF_CONDUCT.md).
- Use clear, descriptive titles and comments in pull requests and issues.
- Participate in discussions constructively.

These norms foster a welcoming and productive community.

### Are there guidelines on coding style and testing?

Yes. GoogleTest/GoogleMock follows Google C++ style guidelines. Contributors should:

- Adhere to [Google's C++ Style Guide](https://google.github.io/styleguide/cppguide.html).
- Write tests ensuring new code works as expected and doesnâ€™t break existing functionality.
- Run all tests locally before submitting.

Following these makes contributions easier to review and maintain.

### Where can I find more detailed contribution instructions?

You can find comprehensive instructions at:

- [CONTRIBUTING.md](https://github.com/google/googletest/blob/main/CONTRIBUTING.md)

## Best Practices for Contributors

- **Start small:** Begin with documentation fixes or small bug patches.
- **Engage early:** Open an issue or discussion if you need help or want feedback.
- **Write clear commit messages:** Make your intentions and context explicit.
- **Test thoroughly:** Include unit tests or update existing ones to cover your changes.
- **Stay in sync:** Pull recent changes often to avoid conflicts.

Following these will make your contributions efficient and well-received.

---

## Common Pitfalls and How to Avoid Them

- **Not running tests locally:** Always ensure tests pass before submission.
- **Ambiguous or incomplete issue descriptions:** Provide clear reproduction steps.
- **Breaking backward compatibility without notice:** Discuss major changes beforehand.
- **Ignoring code style:** Use automated tools or linters to check style.
- **Setting expectations for immediate merges:** Reviews take time; be patient.

Avoiding these will improve your contribution experience.

## Getting Help and Support

- Use the [GitHub Discussions](https://github.com/google/googletest/discussions) and [Issues](https://github.com/google/googletest/issues) pages.
- Join community forums and chat for peer support.
- Review the existing documentation, especially the [Mocking FAQ](https://github.com/google/googletest/blob/main/docs/gmock_faq.md) and [gMock Cookbook](https://github.com/google/googletest/blob/main/docs/gmock_cook_book.md).
- For urgent or complex issues, consider filing a detailed bug report or feature request.
