# Where can I get further help or report an issue?

This page serves as your gateway to various official and community resources where you can seek assistance, report bugs, ask questions, and learn from other active users of GoogleTest and GoogleMock. Whether you are a new user or an experienced developer, knowing where to turn for help is essential in maximizing your experience and effectively solving any problems.

---

## Official Documentation Resources

Leverage the comprehensive official documentation provided by the GoogleTest project to find authoritative, detailed guides and references:

- [GoogleTest User's Guide](https://github.com/google/googletest/tree/main/docs/index.md): Your starting point for understanding and using GoogleTest and GoogleMock effectively.
- [gMock for Dummies](https://google.github.io/googletest/gmock_for_dummies.html): A beginner-friendly tutorial to master creating mock objects and defining expectations.
- [Mocking Reference](https://google.github.io/googletest/reference/mocking.html): In-depth syntax and API details for mocking.
- [gMock Cookbook](https://google.github.io/googletest/gmock_cook_book.html): Practical recipes for common and advanced mocking scenarios.
- [gMock FAQ](https://google.github.io/googletest/gmock_faq.html): Answers to frequently encountered issues and common misunderstandings.

You can also browse through these docs within the repository:

- Explore the docs folder inside the [googletest GitHub repository](https://github.com/google/googletest/tree/main/docs)

## GoogleTest and GoogleMock GitHub Repository

The source project repository serves as a rich resource for help and contributions:

- **Issue Tracker**: Report bugs or request features by opening an issue at [GoogleTest GitHub Issues](https://github.com/google/googletest/issues).
- **Contribution**: See contribution guidelines and how to submit pull requests.
- **Code Examples and Tests**: The `googlemock/test` folder contains tests that demonstrate usage patterns that might help with learning and troubleshooting.
- **Source for Docs**: Documentation files and README markdowns are maintained here, giving you complete context and source.

## Community Support & Discussion Channels

Engage with the broader community to share knowledge, get advice, and hear about best practices:

- **Stack Overflow**: Use the `googletest` and `googlemock` tags to ask and answer questions.
- **Google Groups and Forums**: Official and community-run groups where users and developers discuss issues and features.
- **Mailing Lists**: Join mailing lists related to GoogleTest for announcements and discussions.

## Reporting Issues: Best Practices

When reporting a bug or requesting help, consider the following to facilitate quick resolution:

- Provide a clear, concise description of the problem including expected and actual behavior.
- Include minimal reproducible code snippets or test cases if possible.
- Specify the versions of GoogleTest/GoogleMock used, your compiler, and platform details.
- Mention any error messages or stack traces verbatim.

If you want expedited support or wish to contribute:

- Review the existing open and closed issues before submitting to avoid duplicates.
- Use descriptive titles and label your issues accordingly.

### Practical Tips

- Always ensure you have initialized gMock appropriately in your test executable by calling `InitGoogleMock()` before running tests.
- Use the `--gmock_verbose` flag to increase the verbosity of mock-related debug information when diagnosing unusual behavior.
- Regularly sync with the main repository to stay updated with bug fixes and new features.
