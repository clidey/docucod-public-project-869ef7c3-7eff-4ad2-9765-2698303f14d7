# Where to Find Help

This page directs you to the primary channels and resources for obtaining assistance with GoogleTest and GoogleMock. Whether you are new to the framework, troubleshooting complex issues, or encountering unusual bugs, the following community forums, documentation portals, and official contacts will guide you to timely and effective support.

---

## Official Documentation

Before seeking external help, thoroughly explore the official documentation available on the GoogleTest website and repository. These resources offer comprehensive guides, references, and FAQs designed to resolve most common questions.

- [GoogleTest User's Guide](https://google.github.io/googletest/) — The central hub covering everything from basic usage to advanced mocking techniques.
- [gMock Cheat Sheet](https://google.github.io/googletest/gmock_cheat_sheet.html) — Handy syntax and patterns for mock creation and usage.
- [Legacy gMock FAQ](https://google.github.io/googletest/gmock_faq.html) — Contains solutions for common mocking issues.
- [Mocking Reference](https://google.github.io/googletest/reference/mocking.html) — Detailed API documentation on mocks, expectations, actions, and matchers.

<Tip>
Always start with the official docs to confirm whether your question has already been answered. It saves your time and helps you understand best practices.
</Tip>

## Community Forums and Issue Trackers

Engage with the vibrant GoogleTest community to ask questions, share experiences, or report unusual bugs.

### GitHub Issues

- **GoogleTest GitHub Repository:** https://github.com/google/googletest/issues

Report bugs, request features, or discuss improvements. Make sure to include detailed information about your environment, reproduction steps, and error logs.

### Stack Overflow

- Tag: [`googletest`](https://stackoverflow.com/questions/tagged/googletest) / [`gmock`](https://stackoverflow.com/questions/tagged/gmock)

Ask usage questions or troubleshooting queries. Be specific and provide code snippets or error messages.

### Google Groups and Mailing Lists

- There's no official mailing list, but some community-driven groups exist. Search mailing lists or forums dedicated to C++ testing.

<Info>
Be respectful of community guidelines and search existing topics before posting new questions to reduce duplication.
</Info>

## Tips for Asking Questions Effectively

Clear, concise, and well-formatted questions receive quicker and more precise answers. Here’s how to make the most of community support:

- **Provide minimal complete examples:** Share the relevant portions of your code, especially test classes and mock definitions.
- **Describe your environment:** Platform OS, compiler version, GoogleTest version, and build tools used.
- **Include exact error messages and stack traces:** This helps diagnose problems faster.
- **Explain what you have already tried:** Mention any failed troubleshooting steps to avoid duplicate suggestions.

<Tip>
see the [Mocking for Dummies](https://google.github.io/googletest/gmock_for_dummies.html) guide for introductory concepts before posting.
</Tip>

## Reporting Bugs and Contributing

If you believe you found a bug or want to contribute to GoogleTest, follow these guidelines:

- Confirm your issue is reproducible with the latest release or main branch.
- Check the [changelog](https://github.com/google/googletest/blob/main/CHANGELOG.md) and issues to avoid duplicates.
- Provide minimal reproducible examples to illustrate the problem.
- Follow the contribution guidelines in the repository’s README.

For more details, visit the [Contributing FAQ](https://google.github.io/googletest/faq/contributing.html).
