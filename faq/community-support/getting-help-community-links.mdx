# Getting Help & Community Resources

Helping you succeed with GoogleTest is a crucial part of our mission. This page outlines where to find official support resources, community contributions, and related projects that can enhance your experience. Whether you are troubleshooting, seeking advice, or looking to expand GoogleTest's capabilities, these links and platforms will guide you.

---

## Official Documentation

GoogleTest's official documentation serves as your primary reference for reliable, up-to-date information.

- **GoogleTest User's Guide**: Comprehensive manual with tutorials, best practices, and detailed explanations.
- **Mocking Reference**: In-depth details on creating mocks using GoogleMock.
- **API Reference**: Precise documentation of GoogleTest's core APIs, assertions, matchers, actions, and error handling.
- **Guides**: Practical workflows and recommendations covering core testing patterns, mocking, integration, and optimization.
- **FAQ Section**: Answers to frequently asked questions about usage, troubleshooting, and best practices.

You can access all documentation online through [GoogleTest Documentation](https://google.github.io/googletest/).

---

## Forums and Community Discussions

Engaging with the GoogleTest user community is one of the fastest ways to get help and share knowledge.

- **GitHub Issues**: Use the [GoogleTest GitHub repository](https://github.com/google/googletest/issues) to report bugs, submit feature requests, and participate in ongoing discussions.
- **Stack Overflow**: Post questions tagged with [`googletest`](https://stackoverflow.com/questions/tagged/googletest) or [`gmock`](https://stackoverflow.com/questions/tagged/gmock) for community-driven support.
- **Google Groups**: Join discussions on mailing lists or dedicated groups often linked from the GitHub page.

Community-contributed documentation and insights can be found [here](https://github.com/google/googletest/blob/main/docs/community_created_documentation.md).

## Related Open Source Projects & Tools

Several community and third-party projects extend GoogleTest's capabilities or improve usability:

| Project               | Description                                                                                   | Link                                                                                      |
|-----------------------|-----------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------|
| GTest Runner          | Qt5-based GUI for running GoogleTest binaries on Windows and Linux.                           | [github.com/nholthaus/gtest-runner](https://github.com/nholthaus/gtest-runner)             |
| GoogleTest UI         | C# Windows-based test runner with progress bar and failure inspection.                        | [github.com/ospector/gtest-gbar](https://github.com/ospector/gtest-gbar)                   |
| GTest TAP Listener    | TAP protocol listener enabling integration with test runners supporting TAP output.          | [github.com/kinow/gtest-tap-listener](https://github.com/kinow/gtest-tap-listener)         |
| gtest-parallel        | Parallel test runner to speed up test execution by running tests in parallel.                | [github.com/google/gtest-parallel](https://github.com/google/gtest-parallel)               |
| GoogleTest Adapter    | VS Code extension to view tests in tree format and run/debug from the editor.                | [marketplace.visualstudio.com](https://marketplace.visualstudio.com/items?itemName=DavidSchuldenfrei.gtest-adapter) |
| C++ TestMate          | VS Code extension supporting GoogleTest and Catch2 test views and debugging tools.          | [github.com/matepek/vscode-catch2-test-adapter](https://github.com/matepek/vscode-catch2-test-adapter) |
| Cornichon             | Gherkin-based DSL parser that generates stub test code for GoogleTest.                       | [pypi.org/project/cornichon](https://pypi.org/project/cornichon/)                          |

These tools can help you integrate GoogleTest smoothly into your development environment.

## Reporting Issues and Contributing

If you encounter bugs, unexpected behavior, or have enhancement ideas:

- **Report Issues on GitHub**: Provide detailed descriptions, steps to reproduce, and environment details on the [GitHub issues page](https://github.com/google/googletest/issues).
- **Contributing Code or Documentation**: Follow contribution guidelines in [`CONTRIBUTING.md`](https://github.com/google/googletest/blob/main/CONTRIBUTING.md) to submit pull requests.
- **Providing Feedback**: Share your experience or ask for help via GitHub discussions or mailing lists where available.

Maintainers review community contributions actively to improve the framework continually.

---

## Tips for Effective Community Engagement

- Before posting, **search existing issues and forums** for similar questions to save time.
- When reporting issues, **include reproducible code snippets, build commands, and environment information**.
- Help others by **sharing solutions or workarounds** you discover.
- Respect community guidelines and maintain professional, courteous communication.
