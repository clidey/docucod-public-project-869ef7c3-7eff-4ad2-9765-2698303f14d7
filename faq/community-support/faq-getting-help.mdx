# Getting Help: Community and Support Channels

This guide helps you quickly find the best places to get support, ask questions, or report issues related to GoogleTest and GoogleMock. Whether you’re a beginner needing guidance or an advanced user encountering complex problems, the following resources connect you to community experts, official issue trackers, and key forums.

---

## Frequently Used Support Channels

### 1. Official GitHub Repository Issue Tracker

- **Purpose:** Report bugs, request features, or track the status of known issues.
- **Location:** [https://github.com/google/googletest/issues](https://github.com/google/googletest/issues)
- **Best Practices:**
  - Search existing issues before opening a new one to avoid duplicates.
  - Provide detailed reproduction steps, environment info (OS, compiler, GoogleTest version), and relevant code snippets.
  - Label the issue appropriately (e.g., bug, feature request).

### 2. Google Groups and Mailing Lists

- GoogleTest does not have a dedicated official mailing list, but some unofficial groups exist. However, the **primary community discussions** happen on Stack Overflow and GitHub.

### 3. Stack Overflow

- **Purpose:** General usage questions, best practices, troubleshooting advice.
- **Tags:** Use `googletest` and `googlemock` tag(s) when posting.
- **Link:** [https://stackoverflow.com/questions/tagged/googletest](https://stackoverflow.com/questions/tagged/googletest)
- **Tips for Posting:**
  - Provide clear code examples.
  - Describe errors or unexpected behavior.
  - Tag correctly for visibility.

### 4. Official Documentation and FAQs

- Extensive FAQs address common questions and best practices.
- Browse the [GoogleTest FAQ section](../) or the relevant guides linked below for self-help and faster solutions.

## Getting the Most Out of Support Channels

### Writing Effective Questions and Reports

- **Be concise but complete:** Include your system environment, compiler versions, GoogleTest/GoogleMock versions, and minimal reproducible examples.
- **Explain what you tried:** Show debugging steps, error messages, and configurations.
- **Avoid vague descriptions:** Specify exact error text or unexpected outputs.

### Before Reporting a Bug or Posting a Question

1. Confirm you are using a **supported platform and environment**. See [Prerequisites & System Requirements](/getting-started/setup-and-installation/prerequisites-system-requirements).
2. Review the [Known Issues and Workarounds](/changelog/upgrade-guides/known-issues-workarounds) documentation.
3. Read the [Common Installation and Integration Issues FAQ](/faq/getting-started-troubleshooting/faq-common-installation-issues) to eliminate setup faults.
4. Test the minimal example from the [GoogleTest Primer](../overview/product-intro-core-value/what-is-googletest) or [Writing Your First Test](/getting-started/first-test-experience/writing-your-first-test).

## Practical Tips

- Subscribe or watch GitHub repos to receive notifications on issues and pull requests relevant to your interests.
- Engage respectfully and provide constructive feedback when discussing on forums or issue trackers.
- Use verbose log flags (e.g., `--gmock_verbose=info`) in your tests to gather detailed information for troubleshooting.
