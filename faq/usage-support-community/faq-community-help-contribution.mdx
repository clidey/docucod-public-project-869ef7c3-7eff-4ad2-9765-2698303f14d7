# How to Get Help and Contribute

Connect with the GoogleTest community and access official support channels to troubleshoot issues, report bugs, and contribute improvements. This page guides you through the resources available for assistance and the best practices for submitting code or documentation contributions.

---

## Getting Support

When you encounter issues with GoogleTest or GoogleMock, it’s important to have reliable avenues for assistance. The community and Google maintain a variety of channels to help you resolve problems efficiently.

### Official Support Channels

- **GitHub Issues:** Report bugs, request features, or discuss unexpected behavior at the [GoogleTest GitHub repository](https://github.com/google/googletest/issues). Please provide detailed descriptions and minimal reproducible examples when you file an issue.

- **Stack Overflow:** For usage questions, tagging your queries with `googletest` or `googlemock` can help you get community support from developers familiar with GoogleTest.

- **Mailing Lists & Forums:** While GoogleTest does not maintain a dedicated mailing list, user forums like the Google Testing Blog or related developer communities may provide helpful insights.

### Troubleshooting Tips

Before reaching out for help, consider the following steps to expedite resolution:

1. **Reproduce the Problem in a Minimal Test Case:** Simplify your code to isolate the issue related to GoogleTest.
2. **Consult the Documentation:** Review the [GoogleTest Primer](primer.md), [Mocking Reference](docs/reference/mocking.md), and FAQs for known issues.
3. **Run Tests with Verbose Logging:** Use flags like `--gmock_verbose=info` to get detailed logs.
4. **Check for Environment Issues:** Verify your compiler version and platform against the [Supported Platforms](getting-started/environment-setup/supported-platforms) documentation.

---

## Reporting Bugs

GoogleTest developers appreciate well-documented bug reports that help them identify and fix problems quickly.

### How to Report

- Use the GitHub issue tracker linked above.
- Provide a clear title and description.
- Include your environment details: OS, compiler version, GoogleTest version.
- Attach a minimal reproducing example if possible.
- Include any error messages or logs relevant to the problem.

### What to Avoid

- Do not report issues without first confirming they are reproducible with the latest GoogleTest version.
- Avoid vague statements or incomplete reports; more detail leads to faster fixes.

## Contributing to GoogleTest

GoogleTest welcomes contributions ranging from bug fixes to new features and documentation improvements.

### Contribution Guidelines

- Follow the official [Google Open Source Contribution Guidelines](https://opensource.google/docs/overview/).
- Fork the repository and create a feature branch for your changes.
- Ensure your code is well-tested and follows Google’s C++ style guidelines.
- Include tests and update documentation as appropriate.
- Submit a well-described [pull request](https://github.com/google/googletest/pulls).

### Types of Contributions

- **Bug Fixes:** Address existing issues.
- **Features:** Add new functionality or extend mocking capabilities.
- **Documentation:** Improve clarity, fix typos, or add examples.
- **Tests:** Add tests to improve code coverage.

### Best Practices

- Engage with maintainers on issues before working on large changes.
- Keep commits atomic and descriptive.
- Write clear commit messages and PR descriptions.
- Review and test your changes thoroughly.
