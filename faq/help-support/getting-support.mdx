# Where Can I Get Help?

Discover reliable ways to get assistance when working with GoogleTest and GoogleMock. Whether you are troubleshooting setup issues, seeking usage guidance, or exploring community resources, this page directs you to the most effective support channels and information hubs.

---

## 1. Official Documentation

The primary source of authoritative information on GoogleTest and GoogleMock is the official documentation and repositories:

- **GoogleTest Primer & Tutorials**: Start here to grasp fundamental concepts and best practices.
- **API Reference**: Comprehensive details on testing APIs, assertions, mocking, and matchers.
- **Guides and Advanced Topics**: In-depth tutorials on workflows, fixtures, death tests, mocking, and integration.
- **FAQ Sections**: Answers to common questions, setup troubleshooting, mocking pitfalls, and performance tips.

> You can access the entire documentation navigation and individual pages via the [GoogleTest Documentation Home](https://github.com/google/googletest).


## 2. GitHub Repository

The GitHub repository serves as both the source and an interactive platform for support:

- **Issues**: Report bugs, request features, or ask for help.
- **Discussions**: Engage with the community on usage patterns, challenges, and best practices.
- **Code Examples and Source**: Explore the full implementation to better understand behaviors or to contribute.

Repository URL: [https://github.com/google/googletest](https://github.com/google/googletest)


## 3. Community Platforms

Engage with the developer community to get real-world help, share experiences, and learn:

- **Google Test Mailing List and Forums**: Official forums and discussion lists.
- **Stack Overflow**: Use the `googletest` and `googlemock` tags to find common questions and expert answers.
- **Google Groups**: Participate in 'googletestframework' mailing list for announcements and community support.


## 4. Related Projects and Extensions

Many extensions build on or complement GoogleTest and GoogleMock. Documentation pages and community channels for related projects can provide specialized support:

- Check the [Related Projects and Extensions](https://github.com/google/googletest/tree/main/overview/integration-ecosystem/related-projects-and-extensions) section for links and information.


## 5. Getting Started Guides and Tutorials

For users new to GoogleTest and GoogleMock or those establishing their first tests:

- Follow [Getting Started with Installation and Setup](https://github.com/google/googletest/tree/main/guides/getting-started/installation-and-setup) for platform-specific instructions.
- Consult guides on writing your first test and basic mocking techniques.


## 6. Troubleshooting and Common Issues

If you encounter common build, integration, or runtime problems, these resources are invaluable:

- **Troubleshooting Build and Integration Errors FAQ**: Detailed common errors and fixes.
- **Setup Validation & FAQ**: Step-by-step validation and quick help documentation.


## 7. Contributing and Reporting Issues

If you want to contribute or report issues effectively:

- Review the contributing guidelines and issue templates in the GitHub repository to file clear, actionable reports.
- Use the GitHub issue tracker to monitor existing problems and submit new ones.


---

### Practical Tips for Seeking Support

- When posting questions or issues, include relevant details such as your environment, platform, compiler, build system, and specific error messages.
- Reference specific documentation pages or API versions to clarify your context.
- Search existing issues and discussions before creating new ones to avoid duplication.
