# Where can I get help or report issues?

This page provides clear guidance on the official channels available for support, bug reporting, and community engagement related to GoogleTest and GoogleMock. It also includes valuable links to contribution guidelines, community-contributed documentation, and important open-source projects connected to GoogleTest.

---

## Official Support Channels

If you encounter problems or have questions about GoogleTest or GoogleMock, the following channels are available:

- **GitHub Issues:**
  - Report bugs, submit feature requests, and track ongoing issues at the official [GoogleTest GitHub repository Issues page](https://github.com/google/googletest/issues).
  - Before opening a new issue, please search existing issues to avoid duplicates.

- **Community Discussion:**
  - Engage with other users and developers on forums or community platforms where GoogleTest is discussed.
  - Although GoogleTest itself does not have an official forum, platforms like Stack Overflow and GitHub Discussions in related repositories can be helpful.

- **Contribution Guidelines:**
  - If you want to contribute code, documentation, or fixes, review the [contribution guidelines](https://github.com/google/googletest/blob/main/CONTRIBUTING.md) in the repository.
  - The project welcomes community contributions following the prescribed coding standards and review processes.

## Community-Created Documentation

Apart from official documentation, there are additional resources created by the community that provide insights, tutorials, and examples:

- [Googlemock Insights](https://github.com/ElectricRCAircraftGuy/eRCaGuy_dotfiles/blob/master/googletest/insights.md) by ElectricRCAircraftGuy offers in-depth explanations and hands-on tips, enhancing the understanding of GoogleMock.

These resources can serve as excellent supplementary materials alongside the official docs.

## Contributions and Related Projects

GoogleTest and GoogleMock are part of a broad open-source ecosystem. Here are related projects and areas where you can get involved or explore further:

- **GoogleMock:** Offers comprehensive mocking features integrated tightly with GoogleTest. Learn how to define mock classes, set expectations, and more in the [GoogleMock documentation](https://github.com/google/googletest/tree/main/googlemock).

- **Related Tools & Ecosystem:**
  - Tools for test runners, CI/CD integrations, IDE plugins, and event listeners expanding GoogleTest’s functionality are covered in the ecosystem overview.
  - See the [Related Tools & GoogleTest Ecosystem](https://google.github.io/googletest/overview/integrations/related-tools-ecosystem.html) for details.

- **Community Contributions:**
  - The project encourages community-driven improvements, fixes, samples, and guides to enrich the testing experience for everyone.

## How to Get the Most Out of Support

To receive timely and effective help, keep the following best practices in mind:

1. **Provide Detailed Information:** When reporting issues, include:
   - The version of GoogleTest/GoogleMock you are using.
   - Platform and compiler details.
   - Minimal reproducible example code if possible.
   - Exact error messages or unexpected behavior details.

2. **Use the Right Channels:** Use GitHub Issues for bugs and feature requests. Use Q&A sites like Stack Overflow for general usage questions.

3. **Stay Updated:** Check the [GoogleTest release notes](https://github.com/google/googletest/releases) regularly for fixes and improvements.

4. **Explore Documentation:** Start from the official docs and familiarize yourself with core concepts, common workflows, and best practices to reduce friction.
