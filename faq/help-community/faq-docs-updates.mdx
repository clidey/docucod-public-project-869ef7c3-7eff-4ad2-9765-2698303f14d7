# Documentation Updates and Community Guides

This page provides essential information on where users can find the most current and community-enhanced documentation for GoogleTest and GoogleMock. It also guides users on how to contribute to improving documentation quality, share examples, and access trusted third-party resources.

---

## Accessing the Latest Official Documentation

GoogleTest and GoogleMock documentation is actively maintained in the top-level [`docs`](../../docs) directory of the repository. For the most up-to-date information, examples, and guides, users should refer to this central source.

### Central Documentation Highlights

- **User's Guide:** A comprehensive hub for all things GoogleTest, including primers, advanced topics, and sample code.
- **Mocking Guides:** Detailed instructions for creating and using mocks with `gMock`.
- **API References:** Exhaustive references for all GoogleTest and gMock APIs, including matchers, actions, and mock methods.

Please note that some older documentation content has been migrated here from legacy directories.

## Contributing to Documentation and Examples

Your expertise and experience can help improve GoogleTest's ecosystem. Contributions can include:

- **Enhancements to existing documentation:** Clarify explanations, add examples, or update instructions.
- **New guides or tutorials:** Share workflows, best practices, or niche use-cases.
- **Bug fixes** or **content updates:** Correct inaccuracies or add missing information.

### How to Contribute

1. **Fork the [`googletest` repository](https://github.com/google/googletest).**
2. Make your documentation changes or additions in Markdown format under the `docs/` directory.
3. Follow the contribution guidelines detailed in the repository for pull requests.
4. Provide clear commit messages and detailed descriptions for your changes.

Contributions undergo review to ensure quality and consistency with the rest of the documentation.

## Finding Quality Third-Party Examples and Tutorials

Many open-source projects and developers share example tests and usage patterns online. When looking for third-party content, consider:

- **Reputation of the source:** Prefer established blogs, developer forums, or well-known contributors.
- **Recency:** Software frameworks evolve – use content relevant to the current GoogleTest version.
- **Licensing:** Respect licenses if integrating or redistributing examples.

You may find curated lists or external links on community forums, Stack Overflow, or GitHub repositories devoted to testing best practices.
