# Where to Get Help and Report Issues

This page provides authoritative guidance on the official channels for obtaining help, reporting bugs, and engaging with the GoogleTest and GoogleMock communities. Whether you encounter persistent problems, have advanced usage questions, or want to contribute, knowing where and how to seek support is essential for efficient resolution and collaboration.

---

## 1. Official Support Channels

### GoogleTest GitHub Repository

The primary hub for reporting issues, feature requests, and viewing ongoing development:

- **URL:** [https://github.com/google/googletest](https://github.com/google/googletest)
- Use the **Issues** tab to file bugs or request enhancements.
- Before submitting issues, consult the existing [Issues list](https://github.com/google/googletest/issues) as your problem might have already been reported.
- When reporting, provide clear reproduction steps, environment details (OS, compiler, version), and minimal code snippets if applicable.

### Mailing Lists and Google Groups

Community-driven Q&A and discussions happen here:

- **googletest-users@googlegroups.com** — For usage questions and discussions.
- Archives are searchable and can help you find solutions to common problems.
- Posting detailed questions here often results in quick community assistance.

### Stack Overflow

For peer support with code examples and targeted issues:

- Use the tags `googletest` and `googlemock`.
- Look for highly rated answers to similar questions.
- Ask clear, concise questions illustrating the problem and efforts made.

### Official Documentation

Many common scenarios and troubleshooting steps are documented:

- GoogleTest User Guide: [https://google.github.io/googletest/](https://google.github.io/googletest/)
- Mocking Reference and Recipes for practical examples
- Frequently Asked Questions pages targeting common pain points

## 2. Reporting Issues Effectively

### What to Include When Reporting Bugs

- **Environment details:** OS, compiler name and version, GoogleTest and GoogleMock version
- **Build configuration:** Build system and settings used (CMake, Bazel, etc.)
- **Minimal reproducible example:** The smallest code snippet that triggers the issue
- **Expected behavior vs actual behavior:** Clearly state what you expected and what actually happened
- **Error messages or logs:** Full output including stack traces or warnings
- **Steps to reproduce:** Precise instructions outlining how to recreate the issue

### How to Report

- Create a new issue in the GitHub repository with an informative title and descriptive body
- Label your issue appropriately if required (bug, question, enhancement).
- Follow the repository's issue guidelines if provided.

## 3. Community and Contributing Guidance

### Participation

- Contribute to discussions on GitHub or mailing lists to help others
- Share your knowledge and solutions for common or emerging problems

### Contributing Code or Documentation

- Follow the contribution guidelines in the GitHub repository
- Write clear, maintainable code or documentation adhering to style guides
- Add tests or examples to illustrate new features or fixes

### Staying Updated

- Watch or star the GitHub repository to receive notifications about new releases or changes
- Subscribe to community discussions or mailing lists to stay in the loop
