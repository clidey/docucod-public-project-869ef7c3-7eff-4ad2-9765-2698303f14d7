# Where Can I Find Further Documentation or Get Support?

GoogleTest and GoogleMock offer comprehensive official documentation, active community forums, and dedicated issue tracking systems. This page guides you on the best places to find the latest docs, report issues, access community support, and escalate problems efficiently to keep your testing workflow smooth.

---

## 1. Official Documentation

GoogleTest and GoogleMock maintain an extensive suite of documentation covering from beginner setup to advanced mocking techniques.

- **GoogleTest User's Guide**: Your primary resource for learning test writing, assertions, configuration, and execution workflows.
- **GoogleMock Reference and Tutorials**: For deeply understanding mocking classes, expectations, matchers, and actions.
- **API References**: Detailed listings of all macros, classes, and functions.
- **Guides and Concepts**: Insight into architectural patterns, performance tuning, test structuring, and more.

All documentation is hosted and continuously updated at:

[GoogleTest Documentation Site](https://google.github.io/googletest/)

Users should regularly check this site for new guides, cheat sheets, and cookbook-style examples.

## 2. Issue Reporting and Bug Tracking

If you encounter bugs, missing features, or unexpected behavior in GoogleTest or GoogleMock, the official GitHub repository is the right place to report these issues.

- Visit the [GoogleTest GitHub Issues page](https://github.com/google/googletest/issues) to:
  - Search existing issues to avoid duplicates.
  - Create new issues with detailed descriptions and reproduction steps.

Providing clear, minimal reproduction code snippets helps maintainers address issues more quickly.

## 3. Community Channels and Support Resources

### Forums and Mailing Lists

- The [Google Testing Google Group](https://groups.google.com/g/googletestframework) is an active mailing list where users ask questions, share tips, and discuss best practices.

- Stack Overflow with the [`googletest`](https://stackoverflow.com/questions/tagged/googletest) and [`googlemock`](https://stackoverflow.com/questions/tagged/googlemock) tags is valuable for community-driven Q&A.

### Code Samples and Examples

- The official [googletest samples directory](https://github.com/google/googletest/tree/main/googletest/samples) provides runnable test code illustrating common scenarios and patterns.

### External Learning Resources

- Tutorials such as **gMock for Dummies** and the **Mocking Cookbook** provide approachable learning pathways.

## 4. Escalation and Getting Professional Help

GoogleTest and GoogleMock are open source projects mainly maintained by volunteers and contributors within Google.

- There is no formal commercial support,
- However, some third-party companies offer consulting services.

When filing issues or seeking help:

- Provide clear test code and environment details.
- Review known issues and FAQs for immediate solutions.

For critical bugs affecting your build or deployment pipeline, clearly mark the severity and provide reproduction steps on GitHub.

## 5. Staying Updated

Subscribe to GitHub notifications or RSS feeds for the repository to receive:

- Release announcements
- Bug fixes
- Feature rollouts

Regularly update your local copies of GoogleTest and GoogleMock to incorporate the latest improvements and patches.

---

## Summary

By leveraging official documentation, active community forums, the GitHub issue tracker, and curated example code, you can resolve most questions and issues related to GoogleTest and GoogleMock efficiently. The combination of self-serve resources and community engagement empowers you to maximize value from the frameworks.

## Example: Filing an Issue on GitHub

When reporting an issue:

<Steps>
<Step title="Prepare a Minimal Reproducible Test">
Create a small test program that reliably triggers the issue.
</Step>
<Step title="Gather Environment Details">
Include OS, compiler version, GoogleTest/GoogleMock version, and build configuration.
</Step>
<Step title="Search Existing Issues">
Check if your issue or a similar one has already been reported.
</Step>
<Step title="Open a New Issue">
Submit your reproduction code and environment info on the [github.com/google/googletest/issues](https://github.com/google/googletest/issues) page.
</Step>
<Step title="Engage Respectfully">
Follow up on any questions from maintainers and provide logs or additional info promptly.
</Step>
</Steps>

This approach helps the maintainers help you faster.
