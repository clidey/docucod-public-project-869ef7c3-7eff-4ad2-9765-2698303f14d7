# Where Can I Get Help and Connect with the Community?

This page guides you on how to find official help, report bugs, and engage with the GoogleTest and GoogleMock community. Whether you're seeking assistance with problems, want to report issues, or aim to connect with other users and contributors, this guide directs you to the right resources.

---

## Official Support Channels

### GitHub Repository

The main hub for GoogleTest's source, issues, feature requests, and contributions is its GitHub repository:

- Repository URL: [https://github.com/google/googletest](https://github.com/google/googletest)

Here you can:
- File bug reports in the **Issues** section.
- Search for existing issues and pull requests.
- Track the latest development and releases.
- Contribute code, documentation, or tests.

### Reporting Bugs and Feature Requests

If you encounter bugs or have suggestions for improvements:

1. Visit the [Issues page](https://github.com/google/googletest/issues).
2. Use the existing issue search to avoid duplicates.
3. Create a new issue with a clear, concise title and detailed description including:
   - Steps to reproduce the problem.
   - Expected and actual behavior.
   - Environment details: OS, compiler version, GoogleTest version.
   - Sample code if applicable.

Following these guidelines ensures your report can be quickly understood and acted upon.

## Getting Help

### Community Forums and Discussions

While GoogleTest does not maintain a dedicated official forum, these community-driven platforms offer support and discussions:

- **Stack Overflow**: Tag your questions with `googletest` or `gmock`.
- **Google Groups and mailing lists**: Several groups exist for C++ testing and related topics.
- **Discussion threads on GitHub**: For informal discussions and clarifications.

When asking for help, provide context such as your test setup, mocking approach, and error output to get precise assistance.

### Documentation and Tutorials

Before reaching out, consider consulting the official documentation and tutorials:

- [GoogleTest Primer](primer.md) — Starts you off with easy-to-understand examples.
- [Mocking Reference](docs/reference/mocking.md) — Detailed API documentation for mock objects.
- [Mocking Cookbook](https://google.github.io/googletest/gmock_cook_book.html) — Recipes for common mocking scenarios.
- [GoogleTest FAQ](faq/getting-started-faq/what-is-googletest.mdx) — Answers to frequent questions.

These resources cover common problems, usage patterns, and best practices.

## Third-Party Tools and Extensions

The GoogleTest ecosystem is enriched by community tools that simplify testing:

- IDE plugins for GoogleTest integration.
- Continuous integration helpers.
- Test coverage and analysis tools.

Check community repositories on GitHub and language-specific package managers for such utilities. Always ensure compatibility with the GoogleTest version you use.

## Contributing to the Community

GoogleTest welcomes contributions:

- Submit pull requests for bug fixes, features, or documentation improvements.
- Participate in issue discussions with insights or suggested solutions.
- Share your own tutorials, patterns, or third-party tools.

Please follow the [Contribution Guidelines](https://github.com/google/googletest/blob/main/CONTRIBUTING.md) in the repository for a smooth process.

## Summary

Engaging with the GoogleTest community and seeking help is straightforward thanks to well-established channels and rich documentation. Whether you are reporting an issue, looking for usage advice, or contributing back, the right support is just a few clicks away.
